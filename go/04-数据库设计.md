# 数据库设计 - PostgreSQL 18 完整表结构

## 🎯 设计原则
1. **源文件保护** - 源PDF永远不修改，所有标记分离存储
2. **多语言支持** - 支持中日韩等非字母文字全文搜索
3. **性能优化** - 关键字段建索引，避免N+1查询
4. **审计追踪** - 关键操作记录历史日志（可选）
5. **数据一致性** - 外键约束确保参考完整性
---

## 📋 表结构设计

### 1. 用户表 - users

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    
    -- 认证信息
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    
    -- 用户信息
    display_name VARCHAR(100),
    avatar_url VARCHAR(500),
    
    -- 状态
    is_active BOOLEAN DEFAULT true,
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    last_login_at TIMESTAMP
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_username ON users(username);
```

**说明**：
- email/username 都添加UNIQUE索引用于登录和注册验证
- password_hash 使用bcrypt加密存储（min 60字符）
- is_active 用于逻辑删除（不直接物理删除）

### 2. 文档表 - documents（源文件元数据）

```sql
CREATE TABLE documents (
    id SERIAL PRIMARY KEY,
    
    -- 关联关系
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    
    -- 文档信息
    title VARCHAR(255) NOT NULL,
    description TEXT,
    
    -- 存储信息
    s3_key VARCHAR(500) NOT NULL,           -- S3完整路径
    file_hash VARCHAR(64) NOT NULL,         -- SHA256哈希，防重复上传
    file_size_bytes BIGINT,                 -- 文件大小
    mime_type VARCHAR(50) DEFAULT 'application/pdf',
    
    -- 内容信息
    page_count INTEGER,              -- 总页数
    language VARCHAR(10) DEFAULT 'auto',    -- 文档主要语言
    
  -- 元数据
    is_deleted BOOLEAN DEFAULT false,       -- 逻辑删除
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 唯一约束：同一用户不能重复上传相同文件
    UNIQUE(user_id, file_hash)
);

CREATE INDEX idx_documents_user ON documents(user_id);
CREATE INDEX idx_documents_hash ON documents(file_hash);
CREATE INDEX idx_documents_created ON documents(user_id, created_at DESC);
```

**说明**：
- s3_key：S3中的完整路径，形如 `documents/user_123/doc_456/file.pdf`
- file_hash：用户上传相同PDF时可以检测出来，节省存储
- page_count：由后端在处理PDF时填充
- language：支持用户指定或自动检测（中文、日文、韩文等）
- 注意：**此表只存储元数据，不涉及任何修改操作**

### 3. 文档内容表 - document_content（用于搜索）

```sql
CREATE TABLE document_content (
    id SERIAL PRIMARY KEY,
    
    -- 关联
    document_id INTEGER NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    
    -- 页码信息
    page_num INTEGER NOT NULL,
    
    -- 文本内容
    raw_text TEXT NOT NULL,         -- 原始文本
    
    -- 分词结果（应用层分词）
    segmented_text TEXT,            -- 分词后文本（空格分隔）
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 唯一约束：每文档每页只有一条记录
    UNIQUE(document_id, page_num)
);

-- 关键索引：倒排索引用于全文搜索
CREATE INDEX idx_content_document ON document_content(document_id, page_num);
CREATE INDEX idx_content_tsv ON document_content USING GIN (content_tsv);

-- 可选：word_frequency查询索引
```

**说明**：
- raw_text：从PDF提取的原始文本，保持原样不修改
- segmented_text：使用gse/jieba分词后的结果，词之间用空格分隔
- word_frequency：`{"我": 5, "喜欢": 2}` 格式，用于词频分析
- word_positions：`{"我": [[10,11], [50,51]], "喜欢": [[15,17]]}` 用于高亮
- content_tsv：PG的tsvector格式，用于简单的LIKE搜索

**示例数据**：
```json
{
  "raw_text": "我喜欢阅读PDF文档和笔记",
  "segmented_text": "我 喜欢 阅读 PDF 文档 笔记",
  "word_frequency": {"我": 1, "喜欢": 1, "阅读": 1, "PDF": 1},
  "word_positions": {
    "我": [[0, 1]],
    "喜欢": [[1, 3]],
    "阅读": [[3, 5]],
    "PDF": [[5, 8]],
    "文档": [[8, 10]],
    "笔记": [[11, 13]]
  }
}
```

### 4. 高亮表 - highlights（不修改源文件）

```sql
CREATE TABLE highlights (
    id SERIAL PRIMARY KEY,
    
    -- 关联关系
    document_id INTEGER NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    
    -- 位置信息
    page_num INTEGER NOT NULL,
    
    -- 高亮内容和位置（JSON格式）
    bbox JSONB NOT NULL,                  -- 页面坐标：{x, y, width, height}
    highlighted_text TEXT,              -- 高亮的文本内容
    
    -- 高亮的分词结果（用于搜索）
    segmented_text TEXT,
    color VARCHAR(7) DEFAULT '#FFFF00',   -- 16进制颜色
    
    -- 关联的注释（可选）
    note TEXT,                         -- 用户的注记
    segmented_note TEXT,                    -- 注记分词结果
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- 唯一约束：同一位置的高亮只有一条
    CONSTRAINT unique_highlight UNIQUE(document_id, user_id, page_num, bbox)
);

-- 索引：快速查询某用户的某文档的所有高亮
CREATE INDEX idx_highlights_doc_user ON highlights(document_id, user_id);
CREATE INDEX idx_highlights_page ON highlights(document_id, page_num);

-- 可选：搜索高亮内容
CREATE INDEX idx_highlights_text ON highlights 
    USING GIN (to_tsvector('simple', highlighted_text));
```

**说明**：
- bbox：PDF坐标，形如 `{"x": 100, "y": 200, "width": 50, "height": 15}`
- highlighted_text：用户选中的文本原样存储
- segmented_text：分词后的高亮文本，用于搜索
- note：可选的用户注释
- color：CSS颜色，支持自定义高亮颜色
- 关键：**源PDF文件完全不动，所有数据都在此表**

**示例**：
```json
{
  "document_id": 1,
  "user_id": 100,
  "page_num": 5,
  "bbox": {"x": 100, "y": 200, "width": 150, "height": 20},
  "highlighted_text": "人工智能芯片",
  "segmented_text": "人工智能 芯片",
  "color": "#FFFF00",
  "note": "这是本文的核心概念"
}
```

### 5. 注释表 - annotations（不修改源文件）

```sql
CREATE TABLE annotations (
    id SERIAL PRIMARY KEY,
    
    -- 关联关系
    highlight_id INTEGER REFERENCES highlights(id) ON DELETE SET NULL,
    document_id INTEGER NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    
    -- 位置信息
    page_num INTEGER NOT NULL,
    position JSONB NOT NULL,                -- 注释位置 {x, y}
    
    -- 注释内容
    content TEXT NOT NULL,         -- 注释文本
    segmented_content TEXT,                 -- 分词结果
    
    -- 分类
    type VARCHAR(50) DEFAULT 'comment',     -- comment/question/reminder
    tags TEXT[] DEFAULT ARRAY[]::TEXT[],    -- 标签列表
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_annotations_doc_user ON annotations(document_id, user_id);
CREATE INDEX idx_annotations_page ON annotations(document_id, page_num);
CREATE INDEX idx_annotations_type ON annotations(type);

-- 可选：搜索注释内容
CREATE INDEX idx_annotations_content ON annotations 
    USING GIN (to_tsvector('simple', content));
```

**说明**：
- 可关联高亮（不是必须的）
- position：注释框的位置
- type：分类为评论、问题或提醒
- tags：用户自定义标签，便于分类
- **源PDF文件完全不动**

### 6. 书签表 - bookmarks（不修改源文件）

**说明**：

### 查询用户的所有高亮

```sql
SELECT h.* 
FROM highlights h
WHERE h.user_id = 123 
  AND h.document_id = 456
ORDER BY h.page_num, h.created_at;
```

### 全文搜索用户的标记

不需要专门的 `searchable_marks` 表，使用 `UNION ALL` 联合查询即可满足中小型应用需求：

```sql
SELECT 'content' as type, id, page_num, raw_text as text, ts_rank(content_tsv, query) as score
FROM document_content, to_tsquery('simple', '人工 & 智能') query
WHERE document_id = 456 AND content_tsv @@ query

UNION ALL

SELECT 'highlight' as type, id, page_num, highlighted_text as text, ts_rank(to_tsvector('simple', segmented_text), query) as score
FROM highlights, to_tsquery('simple', '人工 & 智能') query
WHERE document_id = 456 AND user_id = 123 AND to_tsvector('simple', segmented_text) @@ query

UNION ALL

SELECT 'note' as type, id, page_num, content as text, ts_rank(to_tsvector('simple', content), query) as score
FROM annotations, to_tsquery('simple', '人工 & 智能') query
WHERE document_id = 456 AND user_id = 123 AND to_tsvector('simple', content) @@ query

ORDER BY score DESC LIMIT 20;
```

### 导出用户的标记为Markdown

```sql
SELECT 
    d.title,
    h.page_num,
    h.highlighted_text,
    h.note,
    h.created_at
FROM highlights h
JOIN documents d ON h.document_id = d.id
WHERE h.user_id = 123
ORDER BY h.page_num, h.created_at;
```

SELECT * FROM documents

## 📊 性能优化要点

1. **索引策略**
   - 外键字段都要索引（user_id, document_id）
   - 查询条件的前缀列要索引
   - GIN索引用于JSONB和tsvector

2. **避免N+1查询**
   ```sql
   -- ❌ 避免：循环查询
   SELECT * FROM documents WHERE user_id = 123;
   for each doc:
       SELECT * FROM highlights WHERE document_id = doc.id;
   
   -- ✅ 推荐：一次JOIN
   SELECT h.* FROM highlights h
   JOIN documents d ON h.document_id = d.id
   WHERE d.user_id = 123;
   ```

3. **连接池**
   - max_connections: 100（足够用）
   - idle_in_transaction_session_timeout: 10分钟（自动断开空闲)

4. **定期维护**
   ```sql
   -- 每周运行：清理WAL日志
   VACUUM FULL;
   REINDEX;
   
   -- 每天运行：自动清理
   AUTOVACUUM = on;
   ```

---

## 🔐 安全考虑

1. **数据隔离** - 所有查询都带user_id过滤
2. **物理删除 vs 逻辑删除** - documents用逻辑删除（is_deleted）
3. **外键约束** - ON DELETE CASCADE确保数据一致
4. **唯一约束** - 防止重复数据

---
## 总结

这个数据库设计确保：
- ✅ **源文件安全** - PDF永不修改
- ✅ **多语言搜索** - 应用层分词，PG存储
- ✅ **高性能** - 合理的索引和表设计
- ✅ **数据完整** - 外键约束和唯一性保证
- ✅ **易于扩展** - 表结构清晰，易添加新功能

### 8. 自定义词典表 - custom_dictionary (应用层分词支持)

用于存储应用层分词（Jieba/GSE）的动态扩展词汇，支持热更新。

```sql
CREATE TABLE custom_dictionary (
    id SERIAL PRIMARY KEY,
    
    -- 词汇信息
    term VARCHAR(100) NOT NULL UNIQUE, -- 词汇 (e.g., "Transformer")
    frequency INT DEFAULT 0,           -- 词频 (用于Jieba, GSE默认忽略或使用高频)
    nature VARCHAR(20) DEFAULT nz,   -- 词性 (可选, 默认为其他专名)
    
    -- 状态
    is_enable BOOLEAN DEFAULT TRUE,    -- 是否启用
    
    -- 时间戳
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引：加速加载和去重查询
CREATE INDEX idx_dict_term ON custom_dictionary(term);
CREATE INDEX idx_dict_enable ON custom_dictionary(is_enable);
```

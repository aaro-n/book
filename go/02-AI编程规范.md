# AI编程规范 - 确保代码质量和调试效率

## 核心原则

整个项目采用**AI友好的代码分割策略**，确保每个部分都是小的、独立的、易于调试的单元。

---

## 📏 代码规模限制

### 必须遵守的约束

| 指标 | 限制 | 原因 |
|------|------|------|
| 函数行数 | **<30行** | AI理解力强，逻辑清晰 |
| 结构体方法数 | **<5个** | 单一职责，易维护 |
| 文件行数 | **<200行** | 模块化，查看全貌 |
| 嵌套深度 | **<3层** | 逻辑清晰，减少复杂度 |
| 参数个数 | **<5个** | 如需多参数，用结构体 |
| 分支条件 | **<4个** | 如需复杂逻辑，分解函数 |

### ❌ 反面示例（避免）
```go
// 问题1：函数过长（70行）
func ProcessDocument(docID int, userID int, s3client *s3.Client, 
    db *sql.DB, cache *redis.Client, config Config) (map[string]interface{}, error) {
    // 混合了PDF处理、数据库操作、缓存、S3访问...
    // 50+ 行代码，AI难以理解和调试
}

// 问题2：参数过多
func SaveHighlight(docID int, userID int, pageNum int, color string, 
    text string, note string, position interface{}, ...) error {
    // 需要记住6个参数的含义和顺序
}

// 问题3：嵌套过深
if err == nil {
    if response.Code == 0 {
        if len(result) > 0 {
            if value, ok := result[0].(string); ok {
                // 4层嵌套，难以跟踪
            }
        }
    }
}
```

```go
func (s *DocumentService) ProcessNewDocument(ctx context.Context, docID int) error {
    // 1. 从S3获取PDF
    doc, err := s.repo.GetByID(ctx, docID)
    if err != nil {
      return err
    }
    
    // 2. 提取文本
    pdfData, err := s.s3Service.GetObject(ctx, doc.S3Key)
    if err != nil {
        return err
    }
    
    // 3. 分词
    content, err := s.pdfService.ExtractText(pdfData)
    if err != nil {
        return err
    }
    
    // 4. 保存
    return s.repo.SaveContent(ctx, docID, content)
}
```

## 🔍 代码审查清单

在AI生成代码后，请检查以下项目：

- [ ] **行数限制** - 函数<30行，文件<200行
- [ ] **单一职责** - 每个函数做一件事

- [ ] **命名清晰** - 变量和函数名易于理解

- [ ] **类型安全** - 避免interface{}
- [ ] **并发安全** - 如涉及并发，有适当的同步
- [ ] **性能考虑** - 没有明显的性能问题（如N+1查询）
- [ ] **注释完整** - Public函数有godoc注释
- [ ] **测试覆盖** - 核心逻辑有单元测试

## 🚀 AI编程工作流

### 标准流程

1. **需求定义**（您） → 列出功能点和约束
2. **数据模型**（AI） → 生成model.go（<100行）
3. **数据访问**（AI） → 生成repository.go（<120行）
4. **业务逻辑**（AI） → 生成service.go（<150行）
5. **HTTP处理**（AI） → 生成handler.go（<100行）
6. **单元测试**（AI） → 生成_test.go（<80行）
7. **集成测试**（您） → 手工验证端到端流程
8. **代码审查**（您） → 按照检查清单审查

### 逐步迭代

每完成一个小功能就进行：
- 本地测试验证
- 代码审查
- 提交到版本控制

避免一次性实现大功能，容易出错。

### 1. 分层错误处理体系（Go版）

> 来源：Kavita `ExceptionMiddleware.cs` + Komga `Exceptions.kt`，已适配 Go + Gin

#### 1.1 错误类型定义

```go
// internal/util/errors.go

package util

import (
    "database/sql"
    "errors"
    "fmt"
    "net/http"
)

// ---- 错误码 ----
type ErrorCode string

const (
    ErrNotFound       ErrorCode = "NOT_FOUND"
    ErrUnauthorized   ErrorCode = "UNAUTHORIZED"
    ErrForbidden      ErrorCode = "FORBIDDEN"
    ErrValidation     ErrorCode = "VALIDATION_ERROR"
    ErrConflict       ErrorCode = "CONFLICT"
    ErrInternal       ErrorCode = "INTERNAL_ERROR"
)

// ---- AppError ----
type AppError struct {
    Code     ErrorCode `json:"code"`
    Message  string    `json:"message"`
    HTTPCode int       `json:"-"`
    Cause    error     `json:"-"`
}

func (e *AppError) Error() string {
    if e.Cause != nil {
        return fmt.Sprintf("[%s] %s: %v", e.Code, e.Message, e.Cause)
    }
    return fmt.Sprintf("[%s] %s", e.Code, e.Message)
}

func (e *AppError) Unwrap() error { return e.Cause }

// ---- 构造函数 ----
func NewNotFound(msg string) *AppError {
    return &AppError{Code: ErrNotFound, Message: msg, HTTPCode: http.StatusNotFound}
}

func NewUnauthorized(msg string) *AppError {
    return &AppError{Code: ErrUnauthorized, Message: msg, HTTPCode: http.StatusUnauthorized}
}

func NewForbidden(msg string) *AppError {
    return &AppError{Code: ErrForbidden, Message: msg, HTTPCode: http.StatusForbidden}
}

func NewValidation(msg string) *AppError {
    return &AppError{Code: ErrValidation, Message: msg, HTTPCode: http.StatusUnprocessableEntity}
}

func NewConflict(msg string) *AppError {
    return &AppError{Code: ErrConflict, Message: msg, HTTPCode: http.StatusConflict}
}

func NewInternal(msg string, cause error) *AppError {
    return &AppError{Code: ErrInternal, Message: msg, HTTPCode: http.StatusInternalServerError, Cause: cause}
}

// ---- gin 中间件：全局错误处理 ----
func ErrorMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Next()

        if len(c.Errors) > 0 {
            err := c.Errors.Last().Err
            var appErr *AppError
            if errors.As(err, &appErr) {
                c.JSON(appErr.HTTPCode, gin.H{
                    "error": gin.H{"code": appErr.Code, "message": appErr.Message},
                })
            } else {
                c.JSON(http.StatusInternalServerError, gin.H{
                    "error": gin.H{"code": ErrInternal, "message": "内部服务器错误"},
                })
            }
        }
    }
}
```

### 3. HTTP 缓存策略（Go版）

> 来源：Readium Architecture `server/caching.md` + Kavita `BaseApiController.CachedFile()`

| 资源类型 | 策略 | Header 示例 |
|----------|------|------------|
| OPDS Feed / 目录 | ETag + 304 重验证 | `ETag: "abc123"`, `Cache-Control: private, no-cache` |
| PDF 文件下载 | 强缓存 | `Cache-Control: public, max-age=86400` |
| 封面图片 | 强缓存 + ETag | `Cache-Control: public, max-age=86400`, `ETag: "..."` |
| API 响应 | 不缓存或短缓存 | `Cache-Control: private, max-age=60` |

```go
// internal/middleware/cache.go

// ETagMiddleware 为 GET 请求添加 ETag 支持（304 重验证）
func ETagMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        c.Next()

        if c.Request.Method != http.MethodGet || c.Writer.Status() != http.StatusOK {
            return
        }

        // 基于响应体计算 ETag
        if body, ok := c.Get("response_body"); ok {
            hash := sha256.Sum256([]byte(body.(string)))
            etag := fmt.Sprintf("\"%x\"", hash[:16])
            c.Header("ETag", etag)

            if match := c.GetHeader("If-None-Match"); match == etag {
                c.AbortWithStatus(http.StatusNotModified)
            }
        }
    }
}
```

```go
// internal/middleware/auth.go

type AuthUser struct {
    UserID   int    `json:"user_id"`
    Username string `json:"username"`
    Role     string `json:"role"` // "admin" | "user"
}

const AuthUserKey = "auth_user"

// AuthRequired 从 JWT 解析用户信息并注入到 context
func AuthRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        tokenStr := c.GetHeader("Authorization")
        if tokenStr == "" || !strings.HasPrefix(tokenStr, "Bearer ") {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
                "error": gin.H{"code": "UNAUTHORIZED", "message": "未登录或 token 已过期"},
            })
            return
        }

        claims, err := util.ParseJWT(strings.TrimPrefix(tokenStr, "Bearer "))
        if err != nil {
            c.AbortWithStatusJSON(http.StatusUnauthorized, gin.H{
                "error": gin.H{"code": "UNAUTHORIZED", "message": "token 无效"},
            })
            return
        }

        c.Set(AuthUserKey, &AuthUser{
            UserID:   claims.UserID,
            Username: claims.Username,
            Role:     claims.Role,
        })
        c.Next()
    }
}

// GetAuthUser 从 context 中提取 AuthUser
func GetAuthUser(c *gin.Context) *AuthUser {
    user, _ := c.Get(AuthUserKey)
    if u, ok := user.(*AuthUser); ok {
        return u
    }
    return nil
}

// AdminRequired 管理员权限检查中间件
func AdminRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        user := GetAuthUser(c)
        if user == nil || user.Role != "admin" {
            c.AbortWithStatusJSON(http.StatusForbidden, gin.H{
                "error": gin.H{"code": "FORBIDDEN", "message": "需要管理员权限"},
            })
            return
        }
        c.Next()
    }
}
```

```go
func (s *DocumentService) UploadDocument(ctx context.Context, userID int, input UploadInput) (*models.Document, error) {
    uow, err := repository.BeginTx(ctx, s.db)
    if err != nil {
        return nil, util.NewInternal("开启事务失败", err)
    }
    defer uow.Rollback()

        return nil, util.NewInternal("提交事务失败", err)
    }
    return doc, nil
}
```

### 6. 领域事件总线（Go版）

> 来源：Komga `ApplicationEventPublisher` + `DomainEvent.*`

```go
// internal/util/events.go

type DomainEventType string

const (
    EventDocumentUploaded  DomainEventType = "document.uploaded"
    EventDocumentProcessed DomainEventType = "document.processed"
    EventDocumentDeleted   DomainEventType = "document.deleted"
    EventBookmarkCreated   DomainEventType = "bookmark.created"
)

type DomainEvent struct {
    Type      DomainEventType
    Payload   map[string]interface{}
    Timestamp time.Time
}

type EventBus struct {
    subscribers map[DomainEventType][]chan DomainEvent
    mu          sync.RWMutex
}

func NewEventBus() *EventBus {
    return &EventBus{
        subscribers: make(map[DomainEventType][]chan DomainEvent),
    }
}

func (eb *EventBus) Subscribe(eventType DomainEventType) <-chan DomainEvent {
    eb.mu.Lock()
    defer eb.mu.Unlock()
    ch := make(chan DomainEvent, 100)
    eb.subscribers[eventType] = append(eb.subscribers[eventType], ch)
    return ch
}

func (eb *EventBus) Publish(event DomainEvent) {
    eb.mu.RLock()
    defer eb.mu.RUnlock()
    for _, ch := range eb.subscribers[event.Type] {
        select {
        case ch <- event:
        default:
            // 丢弃或记录
        }
    }
}
```

### 7. Lifecycle 生命周期管理（Go版）

> 来源：Komga `BookLifecycle.kt` / `LibraryLifecycle.kt` 命名模式

对复杂实体，将 CRUD + 业务编排聚合到 `*Lifecycle` 中，区别于普通 Service 的单步操作。

```
internal/service/
├── document_service.go      ← 单步操作：get/create/update/delete
├── document_lifecycle.go    ← 完整生命周期：上传→解析→分词→索引→缩略图
├── bookmark_service.go      ← 单步操作
└── search_service.go        ← 搜索协调
```

```go
// internal/service/document_lifecycle.go

type DocumentLifecycle struct {
    docService     *DocumentService
    segmentService *SegmentService
    s3Service      *S3Service
    eventBus       *util.EventBus
}

// ProcessNewDocument 完整上传流程：保存文件 → 提取文本 → 分词 → 存储 → 索引
func (l *DocumentLifecycle) ProcessNewDocument(ctx context.Context, userID int, file *multipart.FileHeader) (*models.Document, error) {
    // 1. 上传到 S3
    s3Key, err := l.s3Service.Upload(ctx, file)
    if err != nil {
        return nil, util.NewInternal("S3上传失败", err)
    }

    // 2. 创建文档记录
    doc, err := l.docService.Create(ctx, userID, file.Filename, s3Key)
    if err != nil {
        return nil, err
    }

    // 3. 异步处理（不阻塞上传响应）
    go func(docID int, key string) {
        if err := l.segmentService.ProcessDocument(context.Background(), docID); err != nil {
            log.Printf("文档处理失败: doc_id=%d, err=%v", docID, err)
            return
        }
        l.eventBus.Publish(util.DomainEvent{
            Type:      util.EventDocumentProcessed,
            Payload:   map[string]interface{}{"doc_id": docID},
            Timestamp: time.Now(),
        })
    }(doc.ID, s3Key)

    return doc, nil
}

// DeleteDocument 完整删除流程：删文件 → 删记录 → 清索引 → 清缓存
func (l *DocumentLifecycle) DeleteDocument(ctx context.Context, docID int) error {
    doc, err := l.docService.GetByID(ctx, docID)
    if err != nil {
        return err
    }

    if err := l.s3Service.Delete(ctx, doc.S3Key); err != nil {
        return util.NewInternal("S3删除失败", err)
    }

    if err := l.docService.Delete(ctx, docID); err != nil {
        return err
    }

    l.eventBus.Publish(util.DomainEvent{
        Type:      util.EventDocumentDeleted,
        Payload:   map[string]interface{}{"doc_id": docID},
        Timestamp: time.Now(),
    })
    return nil
}
```

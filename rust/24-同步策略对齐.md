## 一、核心思路

**离线优先架构**：客户端先存本地，再同步到服务器，服务器作为中央权威。

```
│  SQLite     │ ◄──同步──►   │ PostgreSQL  │

└─────────────┘              └─────────────┘
     ▲                              ▲
     │ 标注读写                     │ 标注读写

```

```rust
// GET /api/v1/sync/annotations?document_id=101&since=2024-01-15T10:00:00Z
async fn get_annotations_since(
    State(pool): State<PgPool>,
    Query(params): Query<SyncQuery>,
) -> Result<Json<Vec<SyncAnnotation>>> {
    let rows = sqlx::query!(
        r#"SELECT w3c_id, w3c_context, locator_href, locator_type,

            target: /* 从 selector_data 还原 */,
            body: /* 从 text_body 还原 */,
            motivation: /* 从字符串解析 */,
            creator: None,
            created: row.created_iso.unwrap_or_default(),
            modified: row.modified_iso,
            device_id: row.device_id,
        }
    }).collect();
    
    Ok(Json(annotations))
```

### 3.2 客户端推送标注

```rust
// POST /api/v1/sync/annotations
async fn push_annotations(
    State(pool): State<PgPool>,
    Json(payload): Json<PushPayload>,
) -> Result<Json<PushResponse>> {
    let mut response = PushResponse {
        accepted: vec![],
        rejected: vec![],
    };
    
    for anno in payload.annotations {
        // 冲突检测：检查 _rev 是否匹配
        let conflict = check_conflict(&pool, &anno.id, &anno.rev).await?;
        
        if conflict {
            response.rejected.push(anno.id.clone());
            continue;
        }
        
        // 写入数据库
        insert_annotation(&pool, &anno).await?;
        response.accepted.push(anno.id.clone());
    }
    
    Ok(Json(response))
}
```

## 四、冲突解决策略

### 4.1 简单方案（MVP）

```
规则1: 最后修改者获胜（modified 时间戳比较）
规则2: 删除优先于修改（deleted=true 时，覆盖所有本地版本）
规则3: 同设备修改不会冲突（deviceId 相同时，直接覆盖）
```

### 4.2 实现逻辑

```rust
async fn resolve_conflict(
    pool: &PgPool,
    incoming: &SyncAnnotation,
    existing: &SyncAnnotation,
) -> ConflictResult {
    // 同设备 → 直接覆盖
    if incoming.device_id == existing.device_id {
        return ConflictResult::Accept;
    }
    
    // 删除优先
    if incoming.deleted == Some(true) {
        return ConflictResult::Accept;
    }
    
    // 最后修改者获胜
    let incoming_time = DateTime::parse_from_rfc3339(&incoming.modified.unwrap_or_default());
    let existing_time = DateTime::parse_from_rfc3339(&existing.modified.unwrap_or_default());
    
    if incoming_time > existing_time {
        ConflictResult::Accept
    } else {
        ConflictResult::Reject
    }
}
```

```sql
CREATE VIEW v_annotation_sync AS
SELECT id, w3c_id, locator_href, selector_data,
       text_body, motivation, device_id,
       updated_at, is_deleted
FROM annotations
WHERE is_deleted = false;

-- 设备追踪表
CREATE TABLE devices (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id),
    device_id VARCHAR(64) NOT NULL,
    device_name VARCHAR(100),
    last_seen TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, device_id)
```

## 七、同步 API 端点清单


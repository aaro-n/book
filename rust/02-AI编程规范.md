# AI编程规范 - 确保代码质量和调试效率

## 核心原则

整个项目采用**AI友好的代码分割策略**，确保每个部分都是小的、独立的、易于调试的单元。

---

## 📏 代码规模限制

### 必须遵守的约束

| 指标 | 限制 | 原因 |
|------|------|------|
| 函数行数 | **<30行** | AI理解力强，逻辑清晰 |
| 结构体方法数 | **<5个** | 单一职责，易维护 |
| 文件行数 | **<200行** | 模块化，查看全貌 |
| 嵌套深度 | **<3层** | 逻辑清晰，减少复杂度 |
| 参数个数 | **<5个** | 如需多参数，用结构体 |
| 分支条件 | **<4个** | 如需复杂逻辑，分解函数 |

### ❌ 反面示例（避免）
```go
// 问题1：函数过长（70行）
func ProcessDocument(docID int, userID int, s3client *s3.Client, 
    db *sql.DB, cache *redis.Client, config Config) (map[string]interface{}, error) {
    // 混合了PDF处理、数据库操作、缓存、S3访问...
    // 50+ 行代码，AI难以理解和调试
}

// 问题2：参数过多
func SaveHighlight(docID int, userID int, pageNum int, color string, 
    text string, note string, position interface{}, ...) error {
    // 需要记住6个参数的含义和顺序
}

// 问题3：嵌套过深
if err == nil {
    if response.Code == 0 {
        if len(result) > 0 {
            if value, ok := result[0].(string); ok {
                // 4层嵌套，难以跟踪
            }
        }
    }
}
```

### ✅ 正确示例（推荐）
```go

func (s *DocumentService) ProcessNewDocument(ctx context.Context, docID int) error {
    // 1. 从S3获取PDF
    doc, err := s.repo.GetByID(ctx, docID)
    if err != nil {
      return err
    }
    
    // 2. 提取文本
    pdfData, err := s.s3Service.GetObject(ctx, doc.S3Key)
    if err != nil {
        return err
    }
    
    // 3. 分词
    content, err := s.pdfService.ExtractText(pdfData)
    if err != nil {
        return err
    }
    
    // 4. 保存
    return s.repo.SaveContent(ctx, docID, content)
}
```

```go

func (h *DocumentHandler) Upload(c *gin.Context) {

        return
    }

    c.JSON(200, gin.H{"document_id": docID})
}
```

## 🔄 错误处理规范

### 清晰的错误定义

```go
// internal/util/errors.go
package util

type ErrorCode string

const (
    ErrDocumentNotFound   ErrorCode = "DOC_NOT_FOUND"
    ErrUnauthorized       ErrorCode = "UNAUTHORIZED"
    ErrInvalidPDF         ErrorCode = "INVALID_PDF"
    ErrS3Upload          ErrorCode = "S3_UPLOAD_FAILED"
    ErrDatabaseError     ErrorCode = "DATABASE_ERROR"
)

type AppError struct {
    Code    ErrorCode
    Message string
    Details string
    Err     error  // 原始错误，便于调试
}

func (e *AppError) Error() string {
    return fmt.Sprintf("[%s] %s: %s", e.Code, e.Message, e.Details)
}

func NewError(code ErrorCode, message string) *AppError {
    return &AppError{
        Code:    code,
        Message: message,
    }
}
```

### 规范的错误处理

```go
// ✅ 好的做法
func GetDocument(ctx context.Context, id int) (*Document, error) {
    doc, err := repo.GetByID(ctx, id)
    if err != nil {
        return nil, NewError(ErrDocumentNotFound, "document not found").WithErr(err)
    }
    return doc, nil
}

// ❌ 避免
func GetDocument(ctx context.Context, id int) (*Document, error) {
    doc, err := repo.GetByID(ctx, id)
    if err != nil {
        // 丢失错误信息，难以调试
        return nil, fmt.Errorf("error")
    }
    return doc, nil
}
```

## 🚀 AI编程工作流

### 标准流程

1. **需求定义**（您） → 列出功能点和约束
2. **数据模型**（AI） → 生成model.go（<100行）
3. **数据访问**（AI） → 生成repository.go（<120行）
4. **业务逻辑**（AI） → 生成service.go（<150行）
5. **HTTP处理**（AI） → 生成handler.go（<100行）
6. **单元测试**（AI） → 生成_test.go（<80行）
7. **集成测试**（您） → 手工验证端到端流程
8. **代码审查**（您） → 按照检查清单审查

### 逐步迭代

每完成一个小功能就进行：
- 本地测试验证
- 代码审查
- 提交到版本控制

避免一次性实现大功能，容易出错。

### 1. 分层错误处理体系

> 来源：Kavita `ExceptionMiddleware.cs` + Komga `Exceptions.kt`

#### 1.1 错误枚举定义

```rust
// src/util/errors.rs

#[derive(Debug, thiserror::Error)]
pub enum AppError {
    // ===== 客户端错误 4xx =====
    #[error("未找到: {0}")]
    NotFound(String),                          // → 404

    #[error("未授权: {0}")]
    Unauthorized(String),                      // → 401

    #[error("禁止访问: {0}")]
    Forbidden(String),                         // → 403

    #[error("验证失败: {0}")]
    Validation(String),                        // → 422

    #[error("资源冲突: {0}")]
    Conflict(String),                          // → 409

    #[error("请求过大: {0}")]
    PayloadTooLarge(String),                   // → 413

    // ===== 服务端错误 5xx =====
    #[error("内部错误: {0}")]
    Internal(String),                          // → 500

    // ===== 自动转换外部错误 =====
    #[error(transparent)]
    Sqlx(#[from] sqlx::Error),                 // → 500
    #[error(transparent)]
    Redis(#[from] redis::RedisError),          // → 500
    #[error(transparent)]
    Io(#[from] std::io::Error),                // → 500
    #[error(transparent)]
    Anyhow(#[from] anyhow::Error),             // → 500
}
```

#### 1.2 HTTP 状态码自动映射

```rust
use axum::{http::StatusCode, response::{IntoResponse, Response}, Json};

impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, error_code, message) = match &self {
            AppError::NotFound(msg)        => (StatusCode::NOT_FOUND,             "NOT_FOUND",        msg.clone()),
            AppError::Unauthorized(msg)    => (StatusCode::UNAUTHORIZED,          "UNAUTHORIZED",     msg.clone()),
            AppError::Forbidden(msg)       => (StatusCode::FORBIDDEN,             "FORBIDDEN",        msg.clone()),
            AppError::Validation(msg)      => (StatusCode::UNPROCESSABLE_ENTITY,  "VALIDATION_ERROR", msg.clone()),
            AppError::Conflict(msg)        => (StatusCode::CONFLICT,              "CONFLICT",         msg.clone()),
            AppError::PayloadTooLarge(msg) => (StatusCode::PAYLOAD_TOO_LARGE,     "PAYLOAD_TOO_LARGE",msg.clone()),
            AppError::Internal(msg)        => (StatusCode::INTERNAL_SERVER_ERROR, "INTERNAL_ERROR",   msg.clone()),
            AppError::Sqlx(e)              => (StatusCode::INTERNAL_SERVER_ERROR, "DB_ERROR",         e.to_string()),
            AppError::Redis(e)             => (StatusCode::INTERNAL_SERVER_ERROR, "CACHE_ERROR",      e.to_string()),
            AppError::Io(e)                => (StatusCode::INTERNAL_SERVER_ERROR, "IO_ERROR",         e.to_string()),
            AppError::Anyhow(e)            => (StatusCode::INTERNAL_SERVER_ERROR, "UNKNOWN_ERROR",    e.to_string()),
        };

        if status.is_server_error() {
            tracing::error!(error_code, %self, "服务器内部错误");
        } else {
            tracing::warn!(error_code, %self, "客户端请求错误");
        }

        let body = serde_json::json!({
            "error": { "code": error_code, "message": message }
        });
        (status, Json(body)).into_response()
    }
}
```

```rust
// src/util/errors.rs

/// 将 sqlx 的 RowNotFound 错误转为 AppError::NotFound，其余为 Internal
pub fn not_found_or_internal<T>(
    result: Result<T, sqlx::Error>,
    entity: &str,
    id: impl std::fmt::Display,
) -> Result<T, AppError> {
    result.map_err(|e| match e {
        sqlx::Error::RowNotFound => AppError::NotFound(format!("{} 不存在: {}", entity, id)),
        other => AppError::Sqlx(other),
    })
}

// Repository 层使用
pub async fn get_by_id(pool: &PgPool, id: i32) -> Result<Document, AppError> {
    not_found_or_internal(
        sqlx::query_as!(Document, "SELECT * FROM documents WHERE id = $1", id)
            .fetch_one(pool).await,
        "文档", id,
    )
}
```

### 2. 统一验证错误响应

> 来源：Komga `ErrorHandlingControllerAdvice.kt`

```rust
// src/util/response.rs

use serde::Serialize;

#[derive(Debug, Serialize)]
pub struct ValidationErrorResponse {
    pub errors: Vec<FieldError>,
}

#[derive(Debug, Serialize)]
pub struct FieldError {
    pub field: String,
    pub message: String,
}

impl From<validator::ValidationErrors> for AppError {
    fn from(e: validator::ValidationErrors) -> Self {
        let errors: Vec<FieldError> = e.field_errors().iter()
            .flat_map(|(field, errs)| {
                errs.iter().map(move |err| FieldError {
                    field: field.to_string(),
                    message: err.message.as_ref()
                        .map(|m| m.to_string())
                        .unwrap_or_else(|| format!("字段 {} 验证失败", field)),
                })
            }).collect();
        AppError::Validation(serde_json::to_string(&ValidationErrorResponse { errors }).unwrap())
    }
}
```

### 3. HTTP 缓存策略

> 来源：Readium Architecture `server/caching.md` + Kavita `BaseApiController.CachedFile()`

| 资源类型 | 策略 | Header 示例 |
|----------|------|------------|
| OPDS Feed / 目录 | ETag + 304 重验证 | `ETag: "abc123"`, `Cache-Control: private, no-cache` |
| PDF 文件下载 | 强缓存 | `Cache-Control: public, max-age=86400` |
| 封面图片 | 强缓存 + ETag | `Cache-Control: public, max-age=86400`, `ETag: "..."` |
| API 响应 | 不缓存或短缓存 | `Cache-Control: private, max-age=60` |

Helper 函数：

```rust
// src/handler/mod.rs

/// 返回物理文件，带 ETag 和 Cache-Control（参考 Kavita）
pub fn cached_file(path: &Path, max_age: u64) -> Result<Response, AppError> {
    if !path.exists() {
        return Err(AppError::NotFound("文件不存在".into()));
    }
    let metadata = std::fs::metadata(path)?;
    let etag = format!("\"{:x}\"", metadata.modified()
        .unwrap().duration_since(std::time::UNIX_EPOCH).unwrap().as_secs());
    let content = std::fs::read(path)?;
    let content_type = mime_guess::from_path(path).first_or_octet_stream().to_string();

    Ok(Response::builder()
        .status(StatusCode::OK)
        .header("Content-Type", content_type)
        .header("ETag", &etag)
        .header("Cache-Control", format!("public, max-age={}", max_age))
        .body(axum::body::Body::from(content)).unwrap())
}
```

```rust
#[axum::async_trait]
impl<S: Send + Sync> FromRequestParts<S> for AuthUser {
    type Rejection = AppError;
    async fn from_request_parts(parts: &mut Parts, _state: &S) -> Result<Self, Self::Rejection> {
        let claims = parts.extensions
            .get::<crate::middleware::auth::JwtClaims>()
            .ok_or_else(|| AppError::Unauthorized("未登录或 token 已过期".into()))?;
        Ok(AuthUser {
            user_id: claims.user_id,
            username: claims.username.clone(),
            role: if claims.is_admin { UserRole::Admin } else { UserRole::User },
        })
    }
}

/// 管理员专用提取器，非管理员自动 403
pub struct AdminUser(pub AuthUser);

#[axum::async_trait]
impl<S: Send + Sync> FromRequestParts<S> for AdminUser {
    type Rejection = AppError;
    async fn from_request_parts(parts: &mut Parts, state: &S) -> Result<Self, Self::Rejection> {
        let user = AuthUser::from_request_parts(parts, state).await?;
        match user.role {
            UserRole::Admin => Ok(AdminUser(user)),
            _ => Err(AppError::Forbidden("需要管理员权限".into())),
        }
    }
}
```

```rust
    let mut rx = state.event_bus.subscribe();
    let pool = state.pool.clone();
    tokio::spawn(async move {
        loop {
            match rx.recv().await {
                Ok(DomainEvent::DocumentProcessed { doc_id }) => {

                        tracing::error!(doc_id, %e, "搜索索引失败");

                Err(_) => break,
```

### 7. Lifecycle 生命周期管理

> 来源：Komga `BookLifecycle.kt` / `LibraryLifecycle.kt` 命名模式

对复杂实体，将 CRUD + 业务编排聚合到 `*Lifecycle` 中，区别于普通 Service 的单步操作。

```
src/service/
├── document_service.rs      ← 单步操作：get/create/update/delete
├── document_lifecycle.rs    ← 完整生命周期：上传→解析→分词→索引→缩略图
├── bookmark_service.rs      ← 单步操作
└── search_service.rs        ← 搜索协调
```

```rust
// src/service/document_lifecycle.rs

pub struct DocumentLifecycle {
    doc_service: Arc<DocumentService>,
    segment_service: Arc<SegmentService>,
    s3_service: Arc<S3Service>,
    event_bus: EventBus,
}

impl DocumentLifecycle {
    /// 完整上传流程：保存文件 → 提取文本 → 分词 → 存储 → 索引
    pub async fn process_new_document(
        &self, user_id: i32, file: UploadedFile,
    ) -> Result<Document, AppError> {
        let s3_key = self.s3_service.upload(&file).await?;
        let doc = self.doc_service.create(user_id, &file, &s3_key).await?;

        // 异步处理，不阻塞上传响应
        let segment_service = self.segment_service.clone();
        let event_bus = self.event_bus.clone();
        let doc_id = doc.id;
        tokio::spawn(async move {
            if let Err(e) = segment_service.process_document(doc_id).await {
                tracing::error!(doc_id, %e, "文档处理失败");
            }
            let _ = event_bus.send(DomainEvent::DocumentProcessed { doc_id });
        });
        Ok(doc)
    }

    /// 完整删除流程：删文件 → 删记录 → 清索引 → 清缓存
    pub async fn delete_document(&self, doc_id: i32) -> Result<(), AppError> {
        let doc = self.doc_service.get_by_id(doc_id).await?;
        self.s3_service.delete(&doc.s3_key).await?;
        self.doc_service.delete(doc_id).await?;
        let _ = self.event_bus.send(DomainEvent::DocumentDeleted { doc_id });
        Ok(())
    }
}
```

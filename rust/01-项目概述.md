# PDF文档管理系统 - 完整项目规划

## 📌 项目目标

构建一个**PDF文档阅读管理系统**，支持中日韩等非英文全文搜索，允许网页和客户端阅读，支持标记同步。

### 核心需求
1. **存储方案**：使用S3存储PDF文件
2. **数据库**：PostgreSQL 18（标准插件，不额外安装）
3. **可选缓存**：Redis（用于热点数据缓存）
4. **全文搜索**：支持中日韩及其他非字母文字
5. **开发模式**：全程使用AI编程，开发人员零经验

---

## 🎯 关键约束条件

### 内存限制
- **Rust 应用专用**：512MB（数据库、Redis不计入）
  - Rust 无 GC，内存占用通常为 Go 的 40-50%，512MB 非常充裕
- **数据库/Redis**：独立配置，不受此限制

### 技术选择
- **后端语言**：Rust 1.80+（零成本抽象，无 GC，编译期内存安全）
- **Web 框架**：Axum 0.7+（基于 Tokio + Tower，类型安全）
- **API 协议**：OPDS 1.2/2.0（发现+搜索+获取）+ Readium Locator（进度）+ W3C Web Annotation（高亮/注释）
- **前端框架**：React + TypeScript（MVP）；Flutter + Rust（Phase 3 客户端）
- **部署方式**：Docker Compose
- **历史参考**：Go 版本文档见 `../go/`，Rust vs Go 对照见 `10-Rust备选方案.md`

### 数据保护
- **源文件不可修改**：用户标记完全分离存储
- **标记表独立**：highlights、annotations、bookmarks 分别存储

## 🚀 快速开始

### 开发准备
```bash
# 1. 启动 Docker 环境
docker-compose up -d

# 2. 初始化数据库
./scripts/init-db.sh

# 3. 启动 Rust 应用
cargo run --release

# 4. 前端开发
cd web && npm run dev
```

### 关键配置位置
- Rust 配置：`config/app.yaml`
- 数据库迁移：`src/db/migrations/`
- 分词配置：`config/app.yaml` 的 `segmenter` 部分
- 搜索引擎：`config/app.yaml` 的 `search_engine` 部分

## 📞 快速参考

| 需求 | 解决方案 |
|------|---------|
| 高内存限制 | Rust 无 GC，空载 ~8MB，+ jieba-rs (~20MB) = 总共 ~30MB，512MB 非常充裕 |
| 第三方阅读器接入 | OPDS 1.2/2.0 提供书目浏览+下载；进度同步通过 KOReader Sync / WebDAV |
| 多端同步 | Readium Locator（进度）+ W3C Web Annotation（高亮/注释）；第三方阅读器走 WebDAV / KOReader Sync |
| AI 编程 | Rust 编译器错误信息精准，编译通过 ≈ 正确运行，降低调试成本 |
| 中文搜索 | GSE + Jieba混用 + Elasticsearch后端 |
| 源文件保护 | 标记表完全分离存储 |
| 灵活搜索 | 后台支持切换ES/Meilisearch/Typesense |
| AI友好代码 | 每个函数<30行，单职责原则 |

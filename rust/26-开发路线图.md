## 二、MVP 阶段（Phase 1，约 4-6 周）

### 2.1 目标

**最小可用版本**：用户上传 PDF → 后端提取文本 → 前端高亮 → 后端保存标注 → 跨设备同步

### 2.2 开发顺序

#### 第 1 周：项目骨架 + 数据模型

```
1. 创建 Cargo 项目
   ├── cargo init pdf-manager
   ├── 配置 Cargo.toml（文档 10）
   └── 创建目录结构（文档 06）

2. 实现数据模型（文档 20 + 23）
   ├── src/models/locator.rs      ← Locator + Locations + LocatorText
   ├── src/models/annotation.rs   ← Annotation + Selector + Body
   └── src/models/mod.rs          ← 模块导出

3. 编写模型单元测试
   ├── test_json_roundtrip()      ← Serde 序列化/反序列化
   ├── test_locator_pdf_page()    ← pdf_page() 辅助方法
   └── test_annotation_valid()    ← W3C 验证（文档 25）
```

#### 第 2 周：数据库 + 基础 API

```
1. 数据库连接（文档 04 + 23）
   ├── src/db/mod.rs              ← PgPool 初始化
   └── migrations/001_init.sql    ← 建表（users, documents, annotations）

2. Repository 层
   ├── src/repository/annotation_repo.rs  ← CRUD
   └── src/repository/document_repo.rs    ← CRUD

3. 基础 API（文档 07）
   ├── src/handler/auth_handler.rs    ← 注册/登录
   ├── src/handler/document_handler.rs ← 上传/列表
   └── src/main.rs                    ← Axum Router 组装
```

#### 第 3 周：PDF 处理

```
1. PDF 文本提取（文档 22）
   ├── src/pdf/extract.rs         ← pdf-extract extract_text_by_pages()
   └── src/pdf/mod.rs             ← 模块导出

2. 分词 + 搜索（文档 05）
   ├── src/util/segment.rs        ← jieba-rs 封装
   └── src/service/search_service.rs ← 搜索逻辑

3. 上传流程整合
   ├── 接收 PDF → S3 存储
   ├── 提取文本 → 存入 document_content
   └── 建立全文索引
```

#### 第 4 周：标注 + 同步

```
1. 标注 API（文档 20 + 25）
   ├── src/handler/annotation_handler.rs  ← CRUD
   ├── 验证逻辑（文档 25）
   └── 集成测试

2. 同步 API（文档 24）
   ├── src/sync/handler.rs        ← ?since= 增量查询
   ├── src/sync/conflict.rs       ← 冲突解决
   └── src/sync/device.rs         ← 设备追踪
```

#### 第 5-6 周：前端 + 联调

```
1. Web 前端 MVP（文档 18）
   ├── PDF.js 阅读器集成
   ├── 高亮选择 + 保存
   └── 标注展示

2. 端到端测试
3. 性能优化
```

# 数据模型实现 — W3C Web Annotation + Readium Locator Serde 定义

> **定位**：本文档是 Rust 项目的**第一份编码蓝图**。在写任何业务逻辑之前，先把数据模型用 Serde 定义好。
> **参考基准**：W3C Web Annotation 规范 → `other/web-annotation/`，Readium Locator → `other/kotlin-toolkit/.../Locator.kt`，Thorium Reader → `other/thorium-reader/`，Komga → `other/komga/.../R2Locator.kt`

## 一、MVP 策略：只取所需

W3C Web Annotation 规范极其庞大，包含 8 种 Selector、多种 State、多目标模式。**MVP 阶段只需要：**

| 需要 | 不需要（留空定义即可） |
|------|------------------------|
| ✅ `TextQuoteSelector` — 纯文本高亮 | ❌ `DataPositionSelector` |
| ✅ `FragmentSelector` — PDF 页码 `page=42` | ❌ `SvgSelector` |
| ✅ `CssSelector` — EPUB DOM 选择 | ❌ `RangeSelector`（两阶段才需要） |
| ✅ `TextualBody` — 文本笔记 | ❌ `Choice` / `Composite` / `List` |
| ✅ `SpecificResource` — 源+选择器 | ❌ `State`（TimeState/HttpRequestState） |
| ✅ Readium `Locator` — 通用位置 | ❌ `LocatorCollection`（Phase 2） |

## 二、Readium Locator — 通用位置模型

> 参考：`other/kotlin-toolkit/readium/shared/src/main/java/org/readium/r2/shared/publication/Locator.kt`
> 参考：`other/komga/komga/src/main/kotlin/org/gotson/komga/domain/model/R2Locator.kt`

### 2.1 核心结构体

```rust
// src/models/locator.rs

use serde::{Deserialize, Serialize};
use std::collections::HashMap;

/// Readium Locator — 出版物中的精确位置。
/// 用于：阅读进度、书签、高亮/注释定位、搜索结果。
///
/// 规范：https://readium.org/architecture/models/locators/
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Locator {
    /// 出版物中资源的 URI（必需）
    /// PDF: "page_042.pdf"、EPUB: "chapter1.xhtml"
    pub href: String,

    /// 资源的 MIME 类型（必需）
    /// PDF: "application/pdf"、EPUB: "application/xhtml+xml"
    #[serde(rename = "type")]
    pub media_type: String,

    /// 可选标题（章节名等）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub title: Option<String>,

    /// 位置表达式（可选但推荐）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub locations: Option<Locations>,

    /// 位置周围的文本上下文（高亮/书签用）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub text: Option<LocatorText>,
}

/// 位置的多种表达方式，至少填一个字段。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Locations {
    /// 片段标识符列表
    /// PDF: ["page=42"]、EPUB: ["#chapter1"]、有声书: ["t=389.84"]
    #[serde(skip_serializing_if = "Vec::is_empty", default)]
    pub fragments: Vec<String>,

    /// 当前资源内的进度 0.0 ~ 1.0
    #[serde(skip_serializing_if = "Option::is_none")]
    pub progression: Option<f64>,

    /// 出版物中的全局位置索引（从 1 开始）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub position: Option<u32>,

    /// 整本出版物的全局进度 0.0 ~ 1.0
    #[serde(skip_serializing_if = "Option::is_none")]
    pub total_progression: Option<f64>,

    /// 扩展位置（HTML: cssSelector/partialCfi/domRange）
    #[serde(skip_serializing_if = "HashMap::is_empty", default)]
    pub other_locations: HashMap<String, serde_json::Value>,
}

/// 定位点周围的文本上下文，用于高亮/书签/搜索摘要。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct LocatorText {
    /// 定位点之前的文本
    #[serde(skip_serializing_if = "Option::is_none")]
    pub before: Option<String>,

    /// 定位点处的文本（被高亮的文本）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub highlight: Option<String>,

    /// 定位点之后的文本
    #[serde(skip_serializing_if = "Option::is_none")]
    pub after: Option<String>,
}
```

### 2.3 各格式 fragments 速查

| 格式 | fragments 示例 | 说明 |
|------|---------------|------|
| **PDF** | `"page=42"` | 物理页码 |
| **PDF 选区** | `"page=42&viewrect=120,340,180,15"` | 页码 + 矩形区域 |
| **EPUB** | `"#chapter1"` | HTML 元素 ID |
| **EPUB 选区** | `"#:~:text=important%20passage"` | 文本片段 |
| **有声书** | `"#t=67"`、`"#t=120,180"` | 秒数/时间段 |

### 2.4 构造辅助方法

```rust
impl Locator {
    /// 为 PDF 创建页码定位器
    pub fn pdf_page(page: u32, total_pages: u32) -> Self {
        let progression = if total_pages > 0 {
            Some(page as f64 / total_pages as f64)
        } else {
            None
        };

        Locator {
            href: format!("page_{:03}.pdf", page),
            media_type: "application/pdf".into(),
            title: None,
            locations: Some(Locations {
                fragments: vec![format!("page={}", page)],
                progression,
                position: Some(page),
                total_progression: progression,
                other_locations: HashMap::new(),
            }),
            text: None,
        }
    }

    /// 为 PDF 创建带选区矩形的定位器（高亮用）
    pub fn pdf_selection(
        page: u32,
        total_pages: u32,
        x: f64, y: f64, w: f64, h: f64,
        highlight_text: &str,
    ) -> Self {
        let fragment = format!(
            "page={}&viewrect={:.0},{:.0},{:.0},{:.0}",
            page, x, y, w, h
        );
        let progression = if total_pages > 0 {
            Some(page as f64 / total_pages as f64)
        } else {
            None
        };

        Locator {
            href: format!("page_{:03}.pdf", page),
            media_type: "application/pdf".into(),
            title: None,
            locations: Some(Locations {
                fragments: vec![fragment],
                progression,
                position: Some(page),
                total_progression: progression,
                other_locations: HashMap::new(),
            }),
            text: Some(LocatorText {
                before: None,
                highlight: Some(highlight_text.into()),
                after: None,
            }),
        }
    }

    /// 获取页码（从 fragments 中解析 "page=N"）
    pub fn page_number(&self) -> Option<u32> {
        self.locations.as_ref()?.fragments.iter()
            .find_map(|f| {
                f.strip_prefix("page=")?
                    .split('&')
                    .next()?
                    .parse::<u32>()
                    .ok()
            })
    }

    /// 获取整书进度 0.0 ~ 1.0
    pub fn total_progress(&self) -> Option<f64> {
        self.locations.as_ref()?.total_progression
    }
}
```

## 三、W3C Web Annotation — 标注数据模型

> 参考：`other/web-annotation/jsonld/anno.jsonld`（JSON-LD 上下文）
> 参考：`other/web-annotation/vocab/wd/ontology/oa.ttl`（本体定义）
> 参考：`other/validate-web-annotation/wadm-schema.js`（验证规则）

### 3.1 核心结构体

```rust
// src/models/annotation.rs

use chrono::{DateTime, Utc};
use serde::{Deserialize, Serialize};

// ============================================================
// 顶层：Annotation
// ============================================================

/// W3C Web Annotation 核心类型。
///
/// 规范：https://www.w3.org/TR/annotation-model/
///
/// 约束（来自 wadm-schema.js 验证规则）：
/// - @context 必须包含 "http://www.w3.org/ns/anno.jsonld"
/// - type 必须是 "Annotation"
/// - target 必须存在
/// - bodyValue 和 body 不能同时存在
/// - id 必须是单个 IRI，不能是数组
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct Annotation {
    /// JSON-LD 上下文（必需）
    #[serde(rename = "@context")]
    pub context: serde_json::Value,

    /// 类型标识（必需，固定 "Annotation"）
    #[serde(rename = "type")]
    pub anno_type: String,

    /// 唯一标识符 IRI
    pub id: String,

    /// 标注目标（必需）：被标注的内容位置
    pub target: AnnotationTarget,

    /// 标注正文（可选）：用户笔记/标签等
    #[serde(skip_serializing_if = "Option::is_none")]
    pub body: Option<AnnotationBody>,

    /// 动机（必需）：为什么创建此标注
    /// MVP 常用值："highlighting"、"bookmarking"、"commenting"
    pub motivation: AnnotationMotivation,

    /// 创建者
    #[serde(skip_serializing_if = "Option::is_none")]
    pub creator: Option<Agent>,

    /// 创建时间 ISO 8601
    pub created: String,

    /// 最后修改时间 ISO 8601
    #[serde(skip_serializing_if = "Option::is_none")]
    pub modified: Option<String>,

    /// 生成此标注的软件
    #[serde(skip_serializing_if = "Option::is_none")]
    pub generator: Option<Agent>,

    /// 版权/许可
    #[serde(skip_serializing_if = "Option::is_none")]
    pub rights: Option<String>,

    /// 文本方向（国际化支持）："auto" | "ltr" | "rtl"
    /// 对阿拉伯语/希伯来语等 RTL 语言重要
    #[serde(skip_serializing_if = "Option::is_none", rename = "textDirection")]
    pub text_direction: Option<TextDirection>,

    /// 处理语言（BCP 47 标签），区分标注内容的处理语言和显示语言
    #[serde(skip_serializing_if = "Option::is_none")]
    pub processing_language: Option<String>,

    /// 规范 IRI：跨系统标注去重用
    #[serde(skip_serializing_if = "Option::is_none")]
    pub canonical: Option<String>,

    /// 来源标注 IRI：标注可引用其他标注（来源追溯）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub via: Option<String>,

    /// 标签数组：用户自定义分类标签
    #[serde(skip_serializing_if = "Vec::is_empty", default)]
    pub tags: Vec<String>,

    /// 剧透标记
    #[serde(skip_serializing_if = "Option::is_none")]
    pub contains_spoiler: Option<bool>,
}

/// 文本方向枚举（W3C Web Annotation 规范）
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum TextDirection {
    Auto,
    Ltr,
    Rtl,
}

// ============================================================
// Target：标注目标
// ============================================================

/// 标注目标，可以是直接 IRI 或 SpecificResource。
/// 使用 untagged enum 让 serde 自动判断。
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(untagged)]
pub enum AnnotationTarget {
    /// 直接指向资源："/docs/101"
    Iri(String),
    /// 完整 SpecificResource（含选择器）
    Specific(Box<SpecificResource>),
}

// ============================================================
// SpecificResource：资源的特定部分
// ============================================================

/// 带选择器的目标资源。
///
/// 约束：source 必需。
///
/// **多选择器支持**：`selector` 字段支持单个或数组。当为 JSON 数组时，
/// 表示同一标注携带多种选择器（冗余策略），客户端按优先级使用。
/// Rust 侧使用 `SelectorList` newtype 包装，serde 自动处理单值/数组。
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct SpecificResource {
    /// 类型标识
    #[serde(rename = "type")]
    pub sr_type: String, // "SpecificResource"

    /// 源资源 IRI（必需）
    pub source: String,

    /// 选择器：细化到资源的某个部分（支持单个或数组）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub selector: Option<SelectorList>,

    /// 样式类：渲染提示（如 "hlt-red"）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub style_class: Option<String>,
}

/// 选择器列表 — 支持单值和数组两种 JSON 表示。
/// 单个选择器 → `Selector::FragmentSelector(...)` 序列化为对象
/// 多个选择器 → `SelectorList::Multiple(vec![...])` 序列化为数组
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(untagged)]
pub enum SelectorList {
    /// 单个选择器（JSON 对象）
    Single(Selector),
    /// 多个选择器（JSON 数组，冗余策略）
    Multiple(Vec<Selector>),
}

// ============================================================
// Selector：选择器（MVP 只需要 FragmentSelector + TextQuoteSelector + CssSelector）
// ============================================================

/// 选择器联合类型。MVP 只实现前三种。
///
/// **多选择器冗余策略**：同一标注可携带多个选择器（`Vec<Selector>`），
/// 提高跨平台兼容性。例如 PDF 标注同时包含 FragmentSelector（页码+坐标）
/// 和 TextQuoteSelector（文本匹配），EPUB 标注同时包含 CssSelector 和
/// TextQuoteSelector。客户端按优先级选择可用选择器。
///
/// 优先级：TextQuoteSelector > FragmentSelector > CssSelector > XPath
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "camelCase")]
pub enum Selector {
    /// PDF 页码定位："page=42"、"page=42&viewrect=..."
    FragmentSelector(FragmentSelector),

    /// 纯文本引号选择器：通过 exact/prefix/suffix 匹配高亮文本
    TextQuoteSelector(TextQuoteSelector),

    /// CSS 选择器：EPUB HTML DOM 定位
    CssSelector(CssSelector),

    /// XPath 选择器：EPUB XML 定位
    #[serde(rename = "XPathSelector")]
    XPath(XPathSelector),

    /// 范围选择器：两个选择器之间的范围（Phase 2）
    RangeSelector(RangeSelector),

    /// 字符位置选择器：start + end 字符偏移（TextQuoteSelector 的版权安全替代）
    TextPositionSelector(TextPositionSelector),

    /// 进度选择器：Thorium 自定义，存储进度百分比
    ProgressionSelector(ProgressionSelector),
}

/// 媒体片段 URI 选择器。
///
/// PDF 示例：`"page=42"`、`"page=42&viewrect=120,340,180,15"`
/// 有声书示例：`"#t=120,180"`
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct FragmentSelector {
    /// 片段值，如 "page=42"（必需）
    pub value: String,

    /// 遵循的媒体片段规范
    /// PDF: "http://tools.ietf.org/rfc/rfc3778"
    /// W3C: "http://www.w3.org/TR/media-frags/"
    #[serde(skip_serializing_if = "Option::is_none")]
    pub conforms_to: Option<String>,
}

/// 文本引用选择器 — MVP 的核心高亮方式。
///
/// 通过精确匹配文本内容定位高亮区域。
/// prefix/suffix 用于消除重复匹配的歧义。
///
/// **版权注意**：对于有版权的内容，TextQuoteSelector 会复制原文，
/// 可考虑使用 TextPositionSelector 替代（仅存储字符偏移）。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TextQuoteSelector {
    /// 精确匹配的文本（必需）
    pub exact: String,

    /// 目标文本之前的上下文（消歧用）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub prefix: Option<String>,

    /// 目标文本之后的上下文（消歧用）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub suffix: Option<String>,

    /// 精化选择器：在此选择器基础上进一步细化（W3C refinedBy 机制）
    /// 例如：TextQuoteSelector → refinedBy → TextPositionSelector
    #[serde(skip_serializing_if = "Option::is_none")]
    pub refined_by: Option<Box<Selector>>,
}

/// CSS 选择器 — EPUB HTML DOM 元素定位。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CssSelector {
    pub value: String,
}

/// XPath 选择器 — EPUB XML 节点定位。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct XPathSelector {
    pub value: String,
}

/// 范围选择器 — 由起始和结束两个选择器界定范围（Phase 2）。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RangeSelector {
    #[serde(rename = "startSelector")]
    pub start_selector: Box<Selector>,
    #[serde(rename = "endSelector")]
    pub end_selector: Box<Selector>,
}

/// 字符位置选择器 — 通过字符偏移定位（TextQuoteSelector 的版权安全替代）。
///
/// W3C 规范：对于有版权的内容，使用位置而非引用文本可避免复制原文。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct TextPositionSelector {
    /// 起始字符偏移（0-based，必需）
    pub start: u64,
    /// 结束字符偏移（0-based，必需）
    pub end: u64,
}

/// 进度选择器 — Thorium Reader 自定义，存储进度百分比。
///
/// 用于在标注中携带进度信息，便于跨平台进度恢复。
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProgressionSelector {
    /// 进度值 0.0 ~ 1.0（必需）
    pub value: f64,
}
```

### 3.3 Annotation 与 Locator 的关系

```
W3C Web Annotation                  Readium Locator
─────────────────                   ───────────────
target → selector                      locations → fragments
  ├─ FragmentSelector("page=42")        ├─ fragments: ["page=42"]
  ├─ TextQuoteSelector(exact="...")     ├─ text → highlight: "..."
  └─ CssSelector(value="...")           └─ other_locations → cssSelector

body → TextualBody(value="笔记")         （Locator 不含笔记，
                                          Annotation 的 body 独立表达）

motivation: "highlighting"              （Locator 不含动机，
                                          Annotation 独立表达）
```

**存储策略**：服务端以 W3C Web Annotation 格式存储，`target.selector` 中嵌套 Locator fragments。前端请求时返回完整的 Annotation JSON-LD，客户端解析 `selector` 还原为 Locator 用于导航。

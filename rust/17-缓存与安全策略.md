# 缓存与安全策略 — Redis 规范、限流、CORS、错误码、CSRF、脱敏

> **设计原则**：在应用层做合理限制，不依赖外部 Nginx/CDN。所有策略面向个人书库场景（日活 <50）。

---

## 一、Redis 缓存规范

### 1.1 Key 命名规范

```
{namespace}:{entity}:{identifier}
```

| 规则 | 示例 |
|------|------|
| 全小写，冒号分隔 | `session:xxx` |
| 不使用特殊字符 | `rate:login:192.168.1.1` |
| 有自然主键用主键 | `presigned:pages/101/042.pdf` |
| 无主键用 UUID/随机值 | `session:kG7xPq2vWm9...` |

### 1.2 已有 Key 汇总

| Key | 值 | TTL | 文档来源 |
|-----|-----|-----|---------|
| `session:{token}` | `{"user_id":1,"username":"alice","role":"user"}` | **7天**，每次请求续期 | `15-认证与安全.md` |
| `user_sessions:{user_id}` | `Set<token>` | **7天** | `15-认证与安全.md`（批量踢出用） |
| `presigned:{s3_key}` | 完整签名 URL | **动态**（≤25天） | `16-S3存储方案.md` |
| `rate:login:{ip}` | 整数计数器 | **15分钟** | `15-认证与安全.md`（登录限流） |

### 1.3 不在 Redis 缓存的数据

| 数据 | 理由 |
|------|------|
| 用户信息 | Session 里已存 `{user_id, username, role}`，无需额外缓存 |
| 文档元数据 | PG 查一次 <1ms，不需要缓存 |
| 搜索热词 | Tantivy/tsvector 自带排序，不需要 Redis |
| 数据库查询结果 | 过早优化。先让 PG 跑，真慢再加 |

### 1.4 Redis 可用性

Redis 是可选服务。代码中所有 Redis 操作使用 `let _ =` 吞错误，不可用时降级到 PG 或其他兜底路径。具体降级策略见各文档章节（Session 见 `15`，签名 URL 见 `16`）。

## 二、CORS 策略

### 2.1 设计

三种客户端场景：

| 客户端 | CORS 影响 | 说明 |
|--------|:---:|------|
| Web 前端（同域部署） | 不需要 | Nginx 反代后同域，浏览器不发 `Origin` |
| Web 前端（开发阶段跨域） | 需要 | `localhost:5173` → `localhost:8080` |
| Flutter / 移动端 | 不影响 | CORS 是浏览器安全机制，移动端不适用 |
| OPDS 客户端 / 脚本 | 不影响 | 服务端间通信，无浏览器 |

### 2.2 配置

```rust
// src/middleware/cors.rs

use tower_http::cors::{CorsLayer, AllowOrigin};

pub fn cors_layer(is_dev: bool) -> CorsLayer {
    if is_dev {
        CorsLayer::new()
            .allow_origin(AllowOrigin::list([
                "http://localhost:5173".parse().unwrap(),  // Vite
                "http://localhost:3000".parse().unwrap(),  // React
                "http://127.0.0.1:5173".parse().unwrap(),
            ]))
            .allow_methods([Method::GET, Method::POST, Method::PUT, Method::DELETE])
            .allow_headers([header::AUTHORIZATION, header::CONTENT_TYPE])
            .allow_credentials(true)
    } else {
        // 生产：同域部署，不需要 CORS；或者按需加白名单
        CorsLayer::permissive() // 简单省事，反正同域不触发
    }
}
```

### 2.3 配置方式

```yaml
# config/app.yaml
cors:
  allowed_origins:
    - "http://localhost:5173"
    - "https://yourdomain.com"
```

默认值：`["http://localhost:5173"]`（开发环境）。生产同域部署时空数组即可。

## 三、Rate Limiting

### 3.1 分层限流

```
┌─────────────────────────────────────────┐
│ L1: tower-governor                      │
│ 全局限流，按 IP，纯内存 token bucket      │
│ 2 req/s，burst 10                       │
│ 所有 /api/* 路径                        │
├─────────────────────────────────────────┤
│ L2: Redis 计数器                        │
│ 登录限流，按 IP                          │
│ 5 次 / 15 分钟                          │
│ 仅 /api/v1/auth/login                  │
└─────────────────────────────────────────┘
```

### 3.2 L1 — 全局限流（tower-governor）

```toml
# Cargo.toml
tower-governor = "0.4"
```

```rust
// src/middleware/rate_limit.rs

use tower_governor::{governor::GovernorConfigBuilder, GovernorLayer, key_extractor::SmartIpKeyExtractor};

pub fn global_rate_limit() -> GovernorLayer<SmartIpKeyExtractor, NoOpMiddleware> {
    let config = GovernorConfigBuilder::default()
        .per_second(2)           // 2 req/s
        .burst_size(10)          // 允许突发 10 个请求
        .finish()
        .unwrap();

    GovernorLayer {
        config: Box::new(config),
    }
}
```

```rust
// main.rs 中挂载
let app = Router::new()
    .merge(api_routes)
    .layer(global_rate_limit());
```

**特点**：
- `SmartIpKeyExtractor`：从 `X-Forwarded-For` / `X-Real-IP` / `SocketAddr` 依次取 IP
- 纯内存 token bucket，不依赖 Redis
- 超限返回 `429 Too Many Requests` + `Retry-After` 头

### 3.3 L2 — 登录限流（Redis）

已在 `15-认证与安全.md` 定义，此处复述：

```rust
// 登录失败时
const MAX_LOGIN_ATTEMPTS: u32 = 5;
const LOGIN_BLOCK_WINDOW: u64 = 900; // 15 分钟

let key = format!("rate:login:{ip}");
let attempts: u32 = redis.incr(&key, 1).await?;
redis.expire(&key, LOGIN_BLOCK_WINDOW).await?;

if attempts > MAX_LOGIN_ATTEMPTS {
    return Err(AppError::TooManyRequests("登录尝试过多，请 15 分钟后再试".into()));
}
```

> L1 防爬虫/滥用，L2 防暴力破解。两层互补，不重复。

## 四、错误码体系

### 4.1 AppError 枚举

```rust
// src/util/errors.rs

use axum::{http::StatusCode, response::{IntoResponse, Response}, Json};
use serde_json::json;
use thiserror::Error;

#[derive(Error, Debug)]
pub enum AppError {
    // 400 — 客户端错误
    #[error("参数校验失败: {0}")]
    BadRequest(String),

    #[error("资源已存在: {0}")]
    Conflict(String),

    // 401/403 — 认证授权
    #[error("未登录或会话已过期")]
    Unauthorized(String),

    #[error("无权限: {0}")]
    Forbidden(String),

    // 404 — 资源不存在
    #[error("资源未找到: {0}")]
    NotFound(String),

    // 422 — 输入校验
    #[error("输入校验失败")]
    ValidationError(#[from] validator::ValidationErrors),

    // 429 — 限流
    #[error("请求过于频繁: {0}")]
    TooManyRequests(String),

    // 500 — 服务端错误
    #[error("内部错误: {0}")]
    Internal(#[from] anyhow::Error),

    // 转发外部库错误
    #[error("数据库错误: {0}")]
    Database(#[from] sqlx::Error),

    #[error("S3 错误: {0}")]
    S3(#[from] aws_sdk_s3::Error),
}
```

### 4.2 HTTP 状态码映射

```rust
impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, code) = match &self {
            AppError::BadRequest(_)       => (StatusCode::BAD_REQUEST, "bad_request"),
            AppError::Conflict(_)         => (StatusCode::CONFLICT, "conflict"),
            AppError::Unauthorized(_)     => (StatusCode::UNAUTHORIZED, "unauthorized"),
            AppError::Forbidden(_)        => (StatusCode::FORBIDDEN, "forbidden"),
            AppError::NotFound(_)         => (StatusCode::NOT_FOUND, "not_found"),
            AppError::ValidationError(_)  => (StatusCode::UNPROCESSABLE_ENTITY, "validation_error"),
            AppError::TooManyRequests(_)  => (StatusCode::TOO_MANY_REQUESTS, "too_many_requests"),
            AppError::Internal(_) |
            AppError::Database(_) |
            AppError::S3(_)               => (StatusCode::INTERNAL_SERVER_ERROR, "internal_error"),
            _                             => (StatusCode::INTERNAL_SERVER_ERROR, "internal_error"),
        };

        let body = json!({
            "error": {
                "code": code,
                "message": self.to_string(),
            }
        });

        (status, Json(body)).into_response()
    }
}
```

### 4.3 Axum 自动错误集成

```rust
// axum Json extractor 反序列化失败 → 自动 422
// validator crate 校验失败 → 自动 422 with details
// sqlx 查询失败 → AppError::Database → 500

// 使用示例
async fn register(
    State(state): State<AppState>,
    Json(payload): Json<RegisterRequest>,  // 反序列化失败 → 422
) -> Result<Json<UserResponse>, AppError> {
    payload.validate()?;                   // validator 校验失败 → 422
    let user = state.auth_service.register(payload).await?; // 业务错误 → AppError
    Ok(Json(user.into()))
}
```

### 5.1 validator crate

```rust
// src/models/user.rs

use validator::Validate;

#[derive(Debug, Deserialize, Validate)]
pub struct RegisterRequest {
    #[validate(email(message = "邮箱格式不正确"))]
    pub email: String,

    #[validate(length(min = 3, max = 30, message = "用户名 3-30 字符"))]
    #[validate(regex(path = *USERNAME_RE, message = "用户名只能包含字母数字下划线"))]
    pub username: String,

    #[validate(length(min = 8, message = "密码至少 8 个字符"))]
    #[validate(custom(function = "validate_password_strength"))]
    pub password: String,

    #[validate(length(max = 100))]
    pub display_name: Option<String>,
}

use lazy_static::lazy_static;
use regex::Regex;

lazy_static! {
    static ref USERNAME_RE: Regex = Regex::new(r"^[a-zA-Z0-9_]+$").unwrap();
}

fn validate_password_strength(password: &str) -> Result<(), validator::ValidationError> {
    let has_letter = password.chars().any(|c| c.is_ascii_alphabetic());
    let has_digit = password.chars().any(|c| c.is_ascii_digit());

    if !has_letter || !has_digit {
        return Err(validator::ValidationError::new("password_weak")
            .with_message("密码必须包含字母和数字".into()));
    }
    Ok(())
}
```

## 六、CSRF 防护

### 6.1 场景分析

```
Web 浏览器登录 → 认证方式: Cookie (SESSION=xxx)
  → 需要 CSRF 防护 ✅ (浏览器自动带 cookie，可能被跨站利用)

Flutter 客户端登录 → 认证方式: Authorization: Bearer xxx
  → 不需要 CSRF 防护 ❌ (无浏览器 cookie，攻击者无法构造)

OPDS / 脚本 → 认证方式: X-API-Key 或 URL 路径
  → 不需要 CSRF 防护 ❌ (服务端间通信)
```

### 6.2 Double Submit Cookie

```rust
// src/middleware/csrf.rs

use axum::{
    extract::Request,
    http::{header, Method, StatusCode},
    middleware::Next,
    response::Response,
};

/// CSRF 中间件：仅检查有 session cookie 的非 GET 请求
pub async fn csrf_protect(
    req: Request,
    next: Next,
) -> Result<Response, StatusCode> {
    // GET/HEAD/OPTIONS — 不检查
    if matches!(req.method(), &Method::GET | &Method::HEAD | &Method::OPTIONS) {
        return Ok(next.run(req).await);
    }

    // 没有 session cookie → 不走 cookie 认证，不检查 CSRF
    if !has_session_cookie(req.headers()) {
        return Ok(next.run(req).await);
    }

    // 有 session cookie → 必须带正确的 CSRF token
    let csrf_cookie = extract_cookie_value(req.headers(), "csrf_token");
    let csrf_header = req.headers()
        .get("X-CSRF-Token")
        .and_then(|v| v.to_str().ok());

    if csrf_cookie.is_none() || csrf_cookie != csrf_header {
        return Err(StatusCode::FORBIDDEN);
    }

    Ok(next.run(req).await)
}

fn has_session_cookie(headers: &header::HeaderMap) -> bool {
    headers.get(header::COOKIE)
        .and_then(|v| v.to_str().ok())
        .map(|c| c.contains("SESSION="))
        .unwrap_or(false)
}

fn extract_cookie_value(headers: &header::HeaderMap, name: &str) -> Option<String> {
    let cookie_str = headers.get(header::COOKIE)?
        .to_str().ok()?;

    for part in cookie_str.split(';') {
        let mut kv = part.trim().splitn(2, '=');
        if kv.next()? == name {
            return kv.next().map(String::from);
        }
    }
    None
}
```

### 6.3 登录时下发 CSRF Token

```rust
// auth_handler.rs 登录成功时

// 生成 CSRF token（32 字节随机 → Base64URL）
let csrf_token = generate_session_token();  // 复用 token 生成函数

// Set-Cookie: csrf_token=xxx; Secure; SameSite=Lax; Path=/
// 注意：非 HttpOnly，浏览器 JS 需要读它
let csrf_cookie = format!(
    "csrf_token={}; Secure; SameSite=Lax; Path=/; Max-Age=604800",
    csrf_token
);
```

**前端使用**：

```javascript
// Web 前端：每个非 GET 请求前读取 csrf_token cookie 并设 header
function getCsrfToken() {
    const match = document.cookie.match(/csrf_token=([^;]+)/);
    return match ? match[1] : '';
}

fetch('/api/v1/documents/upload', {
    method: 'POST',
    headers: {
        'X-CSRF-Token': getCsrfToken(),
        'Content-Type': 'multipart/form-data',
    },
    body: formData,
});
```

> Flutter 客户端不受影响——它不用 cookie，不触发 CSRF 检查。

## 七、日志脱敏

### 7.1 掩码函数

```rust
// src/util/log_mask.rs

/// 密码：完全不记录
pub fn mask_password() -> &'static str {
    "<REDACTED>"
}

/// Token：只记录首 8 位 + 末 4 位，中间截断
/// "kG7xPq2vWm9zL5nR8tY1aB4cD6eF0hJ3xK" → "kG7xPq2v...J3xK"
pub fn mask_token(token: &str) -> String {
    if token.len() <= 12 {
        return "<REDACTED>".into();
    }
    format!("{}...{}", &token[..8], &token[token.len()-4..])
}

/// 邮件：掩码本地部分
/// "alice@example.com" → "a***e@example.com"
pub fn mask_email(email: &str) -> String {
    if let Some((local, domain)) = email.split_once('@') {
        format!("{}@{}", mask_local(local), domain)
    } else {
        "<REDACTED>".into()
    }
}

fn mask_local(local: &str) -> String {
    match local.len() {
        0..=3 => "***".into(),
        n => format!("{}***{}", &local[..1], &local[n-1..]),
    }
}
```

### 7.2 与 tracing 集成

```rust
// 对敏感字段用 #[serde(skip)] 在日志收集中跳过

#[derive(Debug, Deserialize)]
pub struct LoginRequest {
    pub email: String,

    #[serde(skip_serializing)]  // 反序列化时接收，但序列化/日志时不输出
    pub password: String,
}

// 或者手动实现 Debug
impl fmt::Debug for LoginRequest {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        f.debug_struct("LoginRequest")
            .field("email", &mask_email(&self.email))
            .field("password", &mask_password())
            .finish()
    }
}
```

### 7.3 tracing span 过滤

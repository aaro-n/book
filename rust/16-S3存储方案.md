# S3 存储方案 — 签名 URL + CDN + 三级缓存

> **设计原则**：私有桶、签名 URL、CDN 透传。所有文件统一走 302 跳转，不暴露 S3 直连地址。
> **调研参考**：Kavita、Komga、Calibre-Web 三个项目均只支持本地文件系统，无 S3 方案可供参考。本项目是唯一从 Day 1 就选 S3 的书库。

---

## 一、三大项目存储方案调研结论

详见讨论记录，要点如下：

| 项目 | 存储方式 | S3/MinIO | 可借鉴 |
|------|---------|:---:|------|
| Kavita (C#) | 本地磁盘，`IFileSystem` 抽象层 | ❌ | 抽象层思路 |
| Komga (Kotlin) | 本地磁盘，`java.nio.file.Path` | ❌ | 缩略图存 DB BLOB |
| Calibre-Web (Python) | 本地 + Google Drive（无统一接口） | ❌ | 双模式的反面教材 |

**结论**：我们是唯一用 S3 的，没有现成参考。需要自己设计一套完整的存储抽象和缓存体系。

## 二、总体架构

```
┌─────────┐  GET /files/pages/101/042.pdf   ┌──────────┐
│ 客户端   │ ───────────────────────────────►│ 本应用    │
│         │◄──── 302 Location: CDN URL ──────│          │
└────┬────┘                                  └──────────┘
     │                                              │
     │  GET https://<cdn_domain>/pages/              │
     │  101/042.pdf?X-Amz-Signature=...              │
     ▼                                               │
┌──────────┐  缓存未命中   ┌──────────┐
│   CDN    │─────────────►│  S3 源站  │
│ 边缘缓存  │◄──────────────│ (私有桶)  │
│ 7天 TTL  │   验证签名     │          │
└──────────┘              └──────────┘
```

**核心要点**：
- S3 桶**完全私有**，所有访问走签名 URL
- 签名 URL 缓存于 Redis / PG，按需刷新，避免每次请求调 S3 SDK
- CDN 透传完整签名 URL，默认缓存行为（不剥离查询参数）
- 所有文件统一 302 跳转，不区分封面/页面/整本

## 四、签名 URL 三级缓存

### 4.1 缓存层级

```
请求 GET /files/{s3_key}
         │
         ▼
┌──────────────────────────────────────┐
│ L1: Redis                            │
│ key: "presigned:{s3_key}"            │
│ val: 完整签名 URL                     │
│ TTL: 动态（expires_at - 5天）         │
│ 命中 → 302 ✅                        │
│ 不可用/未命中 → L2                   │
└──────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ L2: PostgreSQL                       │
│ SELECT url, expires_at               │
│ FROM presigned_urls WHERE s3_key=?   │
│                                      │
│ remaining > 5天 → 回填 Redis → 302 ✅│
│ remaining ≤ 5天 → 走 L3 刷新         │
│ 未命中 → L3                         │
└──────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────┐
│ L3: S3 SDK presign()                 │
│ 纯本地加密计算，不产生网络请求          │
│ → INSERT/UPDATE PG                   │
│ → SET Redis                          │
│ → 302                               │
└──────────────────────────────────────┘
```

### 4.2 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `S3_SIGN_DURATION` | 30 天 | S3 预签名有效期 |
| `REFRESH_THRESHOLD` | 5 天 | 剩余 ≤ 5 天时触发 S3 SDK 刷新 |
| Redis TTL | **动态** = `remaining - 5天` | 由 PG `expires_at` 决定 |
| PG `expires_at` | `NOW() + 30天` | 签名真实过期时间 |
| PG 定时清理 | 每天 `DELETE WHERE expires_at < NOW()` | 防止表膨胀 |
| CDN 缓存 | 7 天，保留完整 URL（含签名参数） | 零额外配置 |

### 4.3 PG 签名字典表

```sql
CREATE TABLE presigned_urls (
    s3_key VARCHAR(500) PRIMARY KEY,    -- S3 对象路径
    url TEXT NOT NULL,                   -- 完整签名 URL
    expires_at TIMESTAMPTZ NOT NULL,     -- S3 签名过期时间
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_presigned_expires ON presigned_urls(expires_at);
```

**数据量**：1000 本书 × (200 页 + 封面 + 缩略图) ≈ 250,000 行，~150MB。PG 主键索引查找 <1ms。

### 4.4 核心获取逻辑

```rust
use chrono::{Utc, Duration};
use sqlx::PgPool;

#[derive(sqlx::FromRow)]
struct PresignedUrlRow {
    url: String,
    expires_at: chrono::DateTime<Utc>,
}

const S3_SIGN_DURATION: Duration = Duration::days(30);
const REFRESH_THRESHOLD: Duration = Duration::days(5);

impl PresignedUrlCache {
    /// 获取签名 URL（三级缓存：Redis → PG → S3 SDK）
    async fn get(&self, s3_key: &str) -> Result<String, AppError> {
        let redis_key = format!("presigned:{s3_key}");

        // ===== L1: Redis =====
        if let Ok(Some(url)) = self.redis_get(&redis_key).await {
            return Ok(url);
        }
        // Redis 不可用 → 不 panic，走下一层

        // ===== L2: PG =====
        if let Some(row) = self.pg_get(s3_key).await? {
            let remaining = row.expires_at - Utc::now();

            if remaining > REFRESH_THRESHOLD {
                // URL 还新鲜，回填 Redis
                let ttl = (remaining - REFRESH_THRESHOLD).num_seconds();
                let _ = self.redis_set(&redis_key, &row.url, ttl).await;
                return Ok(row.url);
            }
            // remaining <= 5 天 → 太旧，走 L3 重新生成
        }

        // ===== L3: S3 SDK =====
        let (url, expires_at) = self.s3_presign(s3_key, S3_SIGN_DURATION).await?;

        // 写入 PG（upsert）
        sqlx::query!(
            "INSERT INTO presigned_urls (s3_key, url, expires_at) VALUES ($1, $2, $3)
             ON CONFLICT (s3_key) DO UPDATE SET url = $2, expires_at = $3",
            s3_key, url, expires_at
        )
        .execute(&self.pg)
        .await?;

        // 写入 Redis
        let redis_ttl = (S3_SIGN_DURATION - REFRESH_THRESHOLD).num_seconds();
        let _ = self.redis_set(&redis_key, &url, redis_ttl).await;

        Ok(url)
    }
}
```

### 4.5 时间线示例

```
文件: pages/101/042.pdf  内容不变

Day 0:   L3 生成 URL → PG(expires=Day30) → Redis(TTL=Day25)
Day 7:   CDN 缓存过期 → 回源 S3（同一URL）→ 200
Day 14:  CDN 缓存过期 → 回源 S3 → 200
Day 21:  CDN 缓存过期 → 回源 S3 → 200
Day 25:  Redis TTL 到期 → 请求 MISS → L2 PG
         PG: expires=Day30, remaining=5天 ≤ REFRESH_THRESHOLD
         → 触发 L3 刷新！生成新 URL → PG(expires=Day55) → Redis(TTL=Day50)
         → CDN 新缓存条目（新 URL）
Day 30:  旧 URL 在 PG 中不会过期（已 upsert 为新 URL）
Day 55:  下一轮刷新……

S3 SDK 调用：每 30 天 2 次/文件。
Redis 写入：每 30 天 2 次/文件。
```

### 4.6 定时清理

```rust
/// 每天清理过期的签名 URL 行，防止表膨胀
async fn cleanup_expired_urls(pg: &PgPool) -> Result<(), AppError> {
    sqlx::query!("DELETE FROM presigned_urls WHERE expires_at < NOW()")
        .execute(pg)
        .await?;
    Ok(())
}
```

### 4.7 Redis 可用性处理

Redis 是可选服务。代码层面所有 Redis 操作都是 `let _ =`（失败不影响主流程），等价于：

```
Redis 正常 → L1 命中直接返回（最快路径）
Redis 挂了 → L1 MISS → L2 PG（~1ms，用户无感）
Redis 恢复 → 自动回归 L1
```

**不需要健康检测 + 切换逻辑。** PG 始终在线，Redis 挂了自然退化到 L2。

## 五、封面缓存

### 5.1 封面与其他文件的处理差异

| | 封面 | 页面/缩略图/整本 |
|---|---|---|
| 缓存 key | `presigned:cover_url:{doc_id}` | `presigned:{s3_key}` |
| S3 key | `{doc_id}/cover.jpg` | `{doc_id}/pages/001.pdf` 等 |
| 访问端点 | `GET /api/v1/documents/{id}/cover` | `GET /files/{s3_key}` |
| 缓存策略 | 同三级缓存 | 同三级缓存 |
| 签名有效期 | 30 天 | 30 天 |

**封面不走内嵌 base64，统一走 302 + CDN。** 列表 API 返回 `cover_url` 字段，浏览器 `<img>` 并行加载。

## 六、S3 服务封装

```rust
// src/service/s3_service.rs

use aws_sdk_s3::presigning::PresigningConfig;
use chrono::{Utc, Duration};

pub struct S3Service {
    client: aws_sdk_s3::Client,
    bucket: String,
    cdn_base: Option<String>,  // CDN 域名，详见 §9.1；None = 直连 S3
}

impl S3Service {
    /// 生成预签名 URL（GET）
    pub async fn presign_get(&self, s3_key: &str, ttl: Duration) 
        -> Result<String, AppError> 
    {
        let req = self.client
            .get_object()
            .bucket(&self.bucket)
            .key(s3_key);

        let presigned = req
            .presigned(PresigningConfig::expires_in(
                ttl.to_std().unwrap_or(std::time::Duration::from_secs(86400 * 30))
            ))
            .await?;

        let raw_url = presigned.uri().to_string();

        // 如果有 CDN，替换 host 为 CDN 域名
        Ok(match &self.cdn_base {
            Some(cdn) => raw_url.replace(
                &format!("https://{}.s3.amazonaws.com", self.bucket),
                cdn
            ),
            None => raw_url,
        })
    }

    /// 上传对象到 S3
    pub async fn put_object(&self, s3_key: &str, data: &[u8], content_type: &str) 
        -> Result<(), AppError> 
    {
        self.client
            .put_object()
            .bucket(&self.bucket)
            .key(s3_key)
            .body(data.to_vec().into())
            .content_type(content_type)
            .send()
            .await?;
        Ok(())
    }

    /// 删除对象及其签名字典条目
    pub async fn delete_document(&self, doc_id: &str, pg: &PgPool) 
        -> Result<(), AppError> 
    {
        // 删除 S3 中该 document_id 前缀的所有对象
        let objects = self.client
            .list_objects_v2()
            .bucket(&self.bucket)
            .prefix(format!("{doc_id}/"))
            .send()
            .await?;

        if let Some(contents) = objects.contents {
            let mut delete_builder = self.client
                .delete_objects()
                .bucket(&self.bucket);

            let mut objects_to_delete = vec![];
            for obj in &contents {
                if let Some(key) = &obj.key {
                    objects_to_delete.push(
                        aws_sdk_s3::types::ObjectIdentifier::builder()
                            .key(key)
                            .build()?
                    );
                }
            }

            if !objects_to_delete.is_empty() {
                delete_builder.set_delete(Some(
                    aws_sdk_s3::types::Delete::builder()
                        .set_objects(Some(objects_to_delete))
                        .build()?
                ));
                delete_builder.send().await?;
            }
        }

        // 删除 PG 签名字典中相关条目
        sqlx::query!(
            "DELETE FROM presigned_urls WHERE s3_key LIKE $1",
            format!("{doc_id}/%")
        )
        .execute(pg)
        .await?;

        Ok(())
    }
}
```

// 返回所有页面的签名 URL 列表，Flutter 客户端一次性获取缓存 30 天
pub async fn list_page_urls(
    State(state): State<Arc<AppState>>,
    Path(doc_id): Path<String>,
) -> Result<Json<Value>, AppError> {
    let page_count = /* 查 documents 表 */;
    let mut pages = Vec::with_capacity(page_count as usize);

    for num in 1..=page_count {
        let pdf_key = page_pdf_key(&doc_id, num);
        let img_key = page_image_key(&doc_id, num);
        let thumb_key = page_thumb_key(&doc_id, num);

            state.presigned_cache.get(&img_key),
            state.presigned_cache.get(&thumb_key),

            "num": num,
            "pdf_url": pdf_url?,
            "image_url": img_url?,
            "thumb_url": thumb_url?,
        }));
    }

    Ok(Json(json!({
        "document_id": doc_id,
        "page_count": page_count,
        "pages": pages,
        "generated_at": Utc::now().to_rfc3339(),
    })))
}

## 九、CDN 配置

### 9.1 CDN 域名

文档中 `cdn.example.com` 为占位说明，实际各提供商的 CDN 域名如下：

| 存储提供商 | CDN 域名 | 说明 |
|-----------|---------|------|
| **Cloudflare R2** | `{account}.r2.cloudflarestorage.com` | R2 自带全球 CDN，零配置 |
| AWS S3 + CloudFront | `d123abc.cloudfront.net`（自动分配）或自定义域名 CNAME | 需单独配 CloudFront |
| Backblaze B2 + Cloudflare | 自定义域名（如 `files.yourdomain.com`） | Bandwidth Alliance 免流量费 |
| MinIO（自建） | 自己的 VPS IP 或域名 | 无 CDN，直连 |
| BunnyCDN + 任意 S3 | `{name}.b-cdn.net` 或自定义域名 | 纯 CDN 层，任意 S3 做源站 |

**R2 最省事**——它本质是 Cloudflare CDN 边缘节点上的 S3 兼容存储，一个域名同时是存储 endpoint 也是 CDN。不需要额外配置。

**配置方式**（三选一，优先级：环境变量 > app.yaml > 代码默认值）：

```bash
# 方式 1：环境变量（Docker Compose / K8s）
S3_ENDPOINT=https://{account}.r2.cloudflarestorage.com
S3_REGION=auto
S3_BUCKET=book-books
S3_ACCESS_KEY=xxx
S3_SECRET_KEY=xxx
S3_CDN_BASE=https://{account}.r2.cloudflarestorage.com   # CDN 域名；空 = 直连 S3
```

```yaml
# 方式 2：config/app.yaml
s3:
  endpoint: "https://{account}.r2.cloudflarestorage.com"
  region: "auto"
  bucket: "book-books"
  access_key: "xxx"
  secret_key: "xxx"
  cdn_base: "https://{account}.r2.cloudflarestorage.com"  # CDN 域名；空 = 直连 S3
```

### 9.2 配置说明

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 源站 | S3 bucket endpoint | CDN 回源地址 |
| 缓存键 | 完整 URL（含查询字符串） | 默认行为，零额外配置 |
| 缓存 TTL | 7 天 | `Cache-Control: public, max-age=604800` |
| Host 头 | 透传 S3 endpoint | 保证 SigV4 签名验证通过 |

### 9.2 CDN 行为

```
CDN 默认按完整 URL 做缓存键，签名在查询参数里：

  URL_v1: /042.pdf?Signature=AAA  → CDN 缓存条目 #1
  URL_v2: /042.pdf?Signature=BBB  → CDN 缓存条目 #2

S3 签名轮换时，新 URL 形成新 CDN 缓存条目，旧缓存自然过期。
每次签名轮换（每 25 天）最多 1 次 CDN 回源，完全可以接受。
```

### 9.3 可选的 CDN 提供商

| 提供商 | 免费额度 | S3 兼容 | 说明 |
|--------|---------|:---:|------|
| Cloudflare R2 | 10GB 存储 + 无限出站 | ✅ | 零流量费，最佳选择 |
| Backblaze B2 + Cloudflare | 10GB 存储，Bandwidth Alliance 免流量 | ✅ | 需配自定义域名 |
| BunnyCDN | 无免费存储，CDN 低价 | ✅ | 纯 CDN 层 |
| AWS CloudFront | 1TB 免费/月（首年） | ✅ | 原生集成 |

> **推荐**：MVP 用 Cloudflare R2（零流量费 + 全球 CDN 自动启用）。后期流量大了也最便宜。

**S3 SDK 配置（开发/生产切换）**：

// config/app_config.rs
pub fn init_s3(cfg: &AppConfig) -> aws_sdk_s3::Client {
    let config = if cfg.s3.endpoint.contains("minio") 
        || cfg.s3.endpoint.contains("localhost") 
    {
        // 开发环境 MinIO
        aws_sdk_s3::Config::builder()
            .endpoint_url(&cfg.s3.endpoint)
            .force_path_style(true)
            .credentials_provider(
                aws_sdk_s3::config::Credentials::new(
                    &cfg.s3.access_key,
                    &cfg.s3.secret_key,
                    None, None, "minio",
                )
            )
            .region(aws_sdk_s3::config::Region::new("us-east-1"))
            .build()
    } else {
        // 生产 S3 / R2 / B2
        aws_sdk_s3::Config::builder()
            .endpoint_url(&cfg.s3.endpoint)
            .credentials_provider(/* AWS credentials chain */)
            .region(aws_sdk_s3::config::Region::new(&cfg.s3.region))
            .build()
    };
    
    aws_sdk_s3::Client::from_conf(config)
}

## 十一、成本估算

### 11.1 存储成本（以 Cloudflare R2 为例）

```
1000 本书 × 平均 200 页：

  缩略图: 1000 × 200 × 10KB   = 2GB
  预览图: 1000 × 200 × 150KB  = 30GB
  单页PDF: 1000 × 200 × 300KB = 60GB
  整本:   1000 × 50MB         = 50GB
  封面:   1000 × 150KB        = 0.15GB
  ─────────────────────────────────
  总计: ~142GB

Cloudflare R2: 10GB 免费 + 132GB × $0.015 = $1.98/月
Cloudflare R2 出站流量: 免费（无限）
```

### 11.2 S3 SDK 调用成本

```
S3 SDK 的 generate_presigned_url() 是纯本地加密计算，不产生 AWS API 调用。
只有实际 S3 GetObject/PutObject 才收费。

GetObject: $0.0004/千次
  → 1000 本书，每本每 25 天访问一轮，每天 ~40 次
  → 加上 CDN 回源（每 7 天/文件）
  → 每月 <$0.01
```

### 11.3 总成本

| 项目 | 月费 |
|------|------|
| R2 存储 (142GB) | ~$2 |
| R2 出站流量 | $0 |
| S3 API 请求 | <$0.01 |
| **合计** | **<$3/月** |

> 如果用自己的 VPS 跑 MinIO（不用云存储），费用为 $0，但需要自己管理磁盘和备份。

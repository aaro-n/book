## 一、KOReader Sync Protocol

> **参考**：Komga `KoreaderSyncController` + `other/koreader-sync-server/`
> **协议规范**：https://github.com/koreader/koreader-sync-server

### 1.1 协议概述

KOReader 有一套轻量的专属同步 API，使用 **partial MD5 哈希** 匹配书籍：

```
KOReader 客户端                    本项目后端
┌──────────┐                      ┌──────────┐
│ 打开书籍  │                      │          │
│ 计算 MD5  │─── POST /koreader ──→│ MD5 匹配  │
│ (partial) │                      │ → 文档ID  │
│           │←── 返回进度 ──────────│          │
│           │                      │          │
│ 阅读中    │─── PUT 进度 ────────→│ 存储进度  │
└──────────┘                      └──────────┘
```

### 1.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| **POST** | `/koreader/syncs/progress/{document_hash}` | 获取/创建进度 |
| **PUT** | `/koreader/syncs/progress/{document_hash}` | 更新进度 |
| **GET** | `/koreader/syncs/progress/{document_hash}` | 获取进度 |

### 1.3 请求/响应格式

```json

{

```


### 1.4 Partial MD5 哈希

KOReader 使用文件前 1KB 的 MD5 哈希作为文档标识（不是全文件 MD5）：

```rust
// src/service/koreader_service.rs

use md5::Md5;
use md5::Digest;

/// 计算 KOReader 兼容的 partial MD5 哈希
/// 取文件前 1024 字节的 MD5
pub fn koreader_partial_hash(data: &[u8]) -> String {
    let chunk = if data.len() > 1024 { &data[..1024] } else { data };
    let mut hasher = Md5::new();
    hasher.update(chunk);
    format!("{:x}", hasher.finalize())
}

/// 在数据库中通过 partial hash 查找文档
pub async fn find_by_koreader_hash(
    pool: &PgPool,
    hash: &str,
) -> Result<Option<Document>> {
    // documents 表需增加 koreader_hash 字段
    sqlx::query_as!(
        Document,
        "SELECT * FROM documents WHERE koreader_hash = $1 AND is_deleted = false",
        hash
    )
    .fetch_optional(pool)
    .await
}
```

### 1.5 EPUB 进度格式兼容

KOReader 支持两种 EPUB 进度格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| `DocFragment[N]` | `"DocFragment[3]"` | 旧格式，N 为章节序号 |
| `#_doc_fragment_N_` | `"#_doc_fragment_3_"` | 新格式 |

```rust
/// 解析 KOReader 进度字符串
pub fn parse_koreader_progress(progress: &str) -> Option<KoreaderProgress> {
    // 旧格式: DocFragment[3]
    if let Some(rest) = progress.strip_prefix("DocFragment[") {
        if let Some(num_str) = rest.strip_suffix(']') {
            if let Ok(position) = num_str.parse::<u32>() {
                return Some(KoreaderProgress::Epub { position });
            }
        }
    }

    // 新格式: #_doc_fragment_3_
    if let Some(rest) = progress.strip_prefix("#_doc_fragment_") {
        if let Some(num_str) = rest.strip_suffix('_') {
            if let Ok(position) = num_str.parse::<u32>() {
                return Some(KoreaderProgress::Epub { position });
            }
        }
    }

    // PDF: 纯数字页码
    if let Ok(page) = progress.parse::<u32>() {
        return Some(KoreaderProgress::Pdf { page });
    }

    None
}

pub enum KoreaderProgress {
    Pdf { page: u32 },
    Epub { position: u32 },
}
```

## 二、Kobo Sync Protocol

> **参考**：Komga Kobo 兼容层 + Calibre-Web `kobo.py`（完整逆向）

### 2.1 协议概述

Kobo 设备使用专有的同步协议，通过 URL 路径中的 API Key 认证：

```
Kobo 设备                           本项目后端
┌──────────┐                       ┌──────────┐
│ 同步请求  │── GET /kobo/{key}/ ──→│ API Key  │
│           │    v1/library/sync    │ 验证     │
│           │                       │ → 用户ID │
│           │←── 返回变更 ───────────│          │
│           │                       │          │
│ 上传进度  │── PUT /kobo/{key}/ ──→│ koboSpan │
│           │    v1/library/{id}/   │ → R2Locator│
│           │    state              │ 转换     │
└──────────┘                       └──────────┘
```

### 2.2 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| **GET** | `/kobo/{api_key}/v1/library/sync` | 增量同步（SyncPoint） |
| **PUT** | `/kobo/{api_key}/v1/library/{doc_id}/state` | 上传阅读状态 |
| **GET** | `/kobo/{api_key}/v1/library/{doc_id}` | 获取文档元数据 |

### 2.3 koboSpan → R2Locator 转换

Kobo 使用 `koboSpan` 格式定位，需要转换为 Readium Locator：

```rust
// src/service/kobo_service.rs

/// Kobo 阅读状态
#[derive(Debug, Deserialize)]
pub struct KoboReadingState {
    pub reading_progress: KoboProgress,
    pub content_source_progress_percent: f64,
}

#[derive(Debug, Deserialize)]
pub struct KoboProgress {
    pub percent: f64,           // 0.0 ~ 1.0
    pub content_span: String,   // koboSpan 格式
}

/// koboSpan → Readium Locator 转换
///
/// koboSpan 格式: "/body/Document[1]/DocFragment[3]/..."
/// Locator 格式: { href: "...", locations: { fragments: ["page=3"] } }
pub fn kobo_span_to_locator(kobo_span: &str) -> Locator {
    // 解析 koboSpan 中的 DocFragment[N]
    let position = extract_doc_fragment_number(kobo_span)
        .unwrap_or(1);

    Locator {
        href: format!("chapter_{}.xhtml", position),
        media_type: "application/xhtml+xml".into(),
        title: None,
        locations: Some(Locations {
            fragments: vec![format!("#_doc_fragment_{}_", position)],
            progression: None,
            position: Some(position),
            total_progression: None,
            other_locations: HashMap::new(),
        }),
        text: None,
    }
}
```

### 2.4 SyncPoint 增量同步

```rust
/// Kobo SyncPoint 响应
///
/// 参考 Komga `SyncPointLifecycle`：
/// - takeBooksAdded: 新增的书籍
/// - takeBooksChanged: 元数据变更的书籍
/// - takeBooksRemoved: 已删除的书籍
/// - takeBooksReadProgressChanged: 进度变更的书籍
#[derive(Debug, Serialize)]
pub struct KoboSyncResponse {
    pub continuation_token: String,
    pub changes: KoboSyncChanges,
}

#[derive(Debug, Serialize)]
pub struct KoboSyncChanges {
    pub books_added: Vec<KoboBook>,
    pub books_changed: Vec<KoboBook>,
    pub books_removed: Vec<String>,
    pub books_read_progress_changed: Vec<KoboReadProgress>,
}
```

## 三、数据库扩展

### 3.1 documents 表增加 koreader_hash 字段

```sql
ALTER TABLE documents ADD COLUMN koreader_hash VARCHAR(32);
CREATE INDEX idx_documents_koreader_hash ON documents(koreader_hash);
```

### 3.2 Kobo SyncPoint 表

```sql
CREATE TABLE kobo_sync_points (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    continuation_token VARCHAR(64) NOT NULL,
    last_synced_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, continuation_token)
);
```
